# Networking Cheat Sheet

## OSI Model and TCP/IP Stack

| OSI Layer | Name | TCP/IP Layer | Protocols | Devices |
|-----------|------|--------------|-----------|---------|
| 7 | Application | Application | HTTP, DNS, SSH, FTP, SMTP | Load balancer (L7) |
| 6 | Presentation | Application | TLS/SSL, JPEG, ASCII | - |
| 5 | Session | Application | NetBIOS, RPC | - |
| 4 | Transport | Transport | TCP, UDP | Firewall (L4) |
| 3 | Network | Internet | IP, ICMP, ARP | Router |
| 2 | Data Link | Network Access | Ethernet, Wi-Fi, MAC | Switch |
| 1 | Physical | Network Access | Cables, signals, voltages | Hub, NIC |

### TCP vs UDP

| Feature | TCP | UDP |
|---------|-----|-----|
| Connection | Connection-oriented (3-way handshake) | Connectionless |
| Reliability | Guaranteed delivery with ACKs | Best-effort, no guarantee |
| Ordering | Ordered delivery via sequence numbers | No ordering |
| Speed | Slower (overhead) | Faster (minimal overhead) |
| Use cases | HTTP, SSH, FTP, SMTP, databases | DNS, DHCP, video streaming, gaming |

### TCP Three-Way Handshake

```
Client          Server
  │── SYN ──────→│    1. Client sends SYN (seq=x)
  │←── SYN-ACK ──│    2. Server responds SYN-ACK (seq=y, ack=x+1)
  │── ACK ──────→│    3. Client sends ACK (ack=y+1)
  │   Connection Established   │
```

## IP Addressing and CIDR

### Private IP Ranges (RFC 1918)

| Range | CIDR | Class | Addresses |
|-------|------|-------|-----------|
| 10.0.0.0 - 10.255.255.255 | 10.0.0.0/8 | A | 16,777,216 |
| 172.16.0.0 - 172.31.255.255 | 172.16.0.0/12 | B | 1,048,576 |
| 192.168.0.0 - 192.168.255.255 | 192.168.0.0/16 | C | 65,536 |

### Special Addresses

| Address | Purpose |
|---------|---------|
| 127.0.0.0/8 | Loopback (localhost) |
| 169.254.0.0/16 | Link-local (APIPA, auto-assigned) |
| 0.0.0.0 | Unspecified / listen on all interfaces |
| 255.255.255.255 | Broadcast |

| CIDR | Subnet Mask | Total IPs | Usable Hosts |
|------|-------------|-----------|--------------|
| /32 | 255.255.255.255 | 1 | 1 (single host) |
| /30 | 255.255.255.252 | 4 | 2 |
| /28 | 255.255.255.240 | 16 | 14 |
| /27 | 255.255.255.224 | 32 | 30 |
| /26 | 255.255.255.192 | 64 | 62 |

**Formula:** Usable hosts = 2^(32 - prefix) - 2 (subtract network and broadcast addresses).

## DNS Record Types

| Record | Purpose | Example |
|--------|---------|---------|
| A | Maps hostname to IPv4 address | `example.com → 93.184.216.34` |
| AAAA | Maps hostname to IPv6 address | `example.com → 2606:2800:220:1:...` |
| CNAME | Alias to another hostname | `www.example.com → example.com` |
| MX | Mail server for domain | `example.com → mail.example.com (pri 10)` |
| TXT | Arbitrary text (SPF, DKIM, verification) | `v=spf1 include:_spf.google.com ~all` |
| NS | Nameserver for zone | `example.com → ns1.example.com` |
| SOA | Start of authority (zone info) | Serial, refresh, retry, expire, TTL |
| PTR | Reverse DNS (IP to hostname) | `34.216.184.93 → example.com` |
| SRV | Service location (port + host) | `_sip._tcp.example.com → sip.example.com:5060` |
| CAA | Certificate authority authorization | `example.com CAA 0 issue "letsencrypt.org"` |

### DNS Resolution Order

```
Browser cache → OS cache (/etc/hosts) → Recursive Resolver
  → Root servers (.) → TLD servers (.com) → Authoritative nameserver
  → IP address returned and cached at each level
```

## Key Networking Commands

### Connectivity Testing

```bash
# Ping (ICMP echo)
ping -c 4 google.com                        # 4 pings
ping -c 1 -W 2 10.0.0.1                     # 1 ping, 2s timeout

# Traceroute (show path to destination)
traceroute google.com
tracepath google.com                          # No root needed
mtr google.com                                # Continuous traceroute
```

### DNS Queries

```bash
# dig (most detailed)
dig example.com                               # Default A record
dig example.com MX                            # Mail records
dig example.com +short                        # Brief output
dig @8.8.8.8 example.com                     # Query specific server
dig -x 93.184.216.34                          # Reverse lookup
dig example.com +trace                        # Full resolution trace

# nslookup (simpler)
nslookup example.com
nslookup -type=MX example.com

# host (simplest)
host example.com
```

### HTTP Requests

```bash
# curl
curl -v https://example.com                   # Verbose (headers + body)
curl -sS -o /dev/null -w "%{http_code}" URL   # Status code only
curl -X POST -H "Content-Type: application/json" \
     -d '{"key":"value"}' URL                  # POST JSON
curl -I URL                                    # HEAD request (headers)
curl -L URL                                    # Follow redirects
curl -u user:pass URL                          # Basic auth
curl --retry 3 --retry-delay 5 URL            # Retry on failure

# wget
wget -q URL -O output.file                    # Quiet download
wget --spider URL                              # Check if URL exists
```

### Socket and Connection Inspection

```bash
# ss (modern replacement for netstat)
ss -tlnp                     # Listening TCP ports + process names
ss -ulnp                     # Listening UDP ports + process names
ss -s                        # Socket statistics summary
ss -tn state established     # Established connections
ss -tn dst 10.0.0.5          # Connections to specific host

# netstat (legacy)
netstat -tlnp                # Same as ss -tlnp
netstat -rn                  # Routing table
```

### Packet Capture

```bash
# tcpdump
tcpdump -i eth0                               # Capture on interface
tcpdump -i eth0 port 80                       # Filter by port
tcpdump -i eth0 host 10.0.0.5                 # Filter by host
tcpdump -i eth0 -w capture.pcap               # Save to file
tcpdump -i eth0 'tcp port 443 and host 10.0.0.5'  # Combined filter
tcpdump -r capture.pcap                       # Read saved capture
```

### IP and Routing

```bash
ip addr show                  # Show all interfaces
ip addr show eth0             # Specific interface
ip route show                 # Show routing table
ip route get 8.8.8.8          # Which route for destination
ip link set eth0 up/down      # Enable/disable interface
ip neigh show                 # ARP table (neighbor cache)
```

## Firewall Rules

### ufw (Uncomplicated Firewall)

```bash
ufw enable                                    # Enable firewall
ufw default deny incoming                     # Default policy
ufw default allow outgoing                    # Default policy
ufw allow 22/tcp                              # Allow SSH
ufw allow from 10.0.0.0/24 to any port 5432  # Allow Postgres from subnet
ufw deny 3306                                 # Block MySQL
ufw status verbose                            # Show rules
ufw delete allow 80/tcp                       # Remove rule
```

### iptables

```bash
# View rules
iptables -L -n -v                             # List all rules (numeric)

# Allow established connections
iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

# Allow SSH
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Allow HTTP/HTTPS
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Drop everything else
iptables -A INPUT -j DROP

# Save rules (varies by distro)
iptables-save > /etc/iptables.rules
```

## Troubleshooting Methodology

### Layer-by-Layer Approach (Bottom-Up)

```
1. Physical/Link:  Is the interface up?           ip link show
2. Network:        Do we have an IP?              ip addr show
3. Gateway:        Can we reach the gateway?      ping <gateway-ip>
4. Internet:       Can we reach external IPs?     ping 8.8.8.8
5. DNS:            Can we resolve names?           dig example.com
6. Application:    Can we reach the service?       curl http://example.com
7. Port:           Is the service listening?       ss -tlnp | grep :80
```

### Common Error Diagnosis

| Symptom | Likely Cause | Commands |
|---------|--------------|----------|
| "Connection refused" | Service not running on that port | `ss -tlnp`, `systemctl status` |
| "Connection timed out" | Firewall dropping packets / host unreachable | `ping`, `traceroute`, `ufw status` |
| "Name resolution failed" | DNS misconfiguration | `dig`, `cat /etc/resolv.conf` |
| "No route to host" | Routing issue / interface down | `ip route`, `ip link show` |
| Intermittent connectivity | Packet loss / network congestion | `mtr`, `ping -c 100` |
| Slow connections | Bandwidth / latency / DNS delays | `mtr`, `dig`, `curl -w "%{time_total}"` |

### Quick Health Checks

```bash
# Is the service listening?
ss -tlnp | grep :8080

# Can I reach the port?
nc -zv hostname 8080

# What is my public IP?
curl -s ifconfig.me

# Test TCP connection
curl -v telnet://hostname:port

# Check SSL certificate
openssl s_client -connect example.com:443 -brief

# Measure response time
curl -o /dev/null -s -w "DNS: %{time_namelookup}s\nConnect: %{time_connect}s\nTTFB: %{time_starttransfer}s\nTotal: %{time_total}s\n" URL
```
