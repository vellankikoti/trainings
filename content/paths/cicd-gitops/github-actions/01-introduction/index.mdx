<Callout type="story" title="The Story">
Before GitHub Actions, setting up CI/CD required third-party services — Jenkins servers to maintain, CircleCI accounts to manage, webhook integrations to configure. In November 2019, GitHub launched Actions, bringing CI/CD directly into the platform where your code already lives. No external servers. No separate accounts. Just a YAML file in your repository, and your builds, tests, and deployments run automatically on GitHub's infrastructure. Within two years, it became the most popular CI/CD platform for open-source projects.
</Callout>

## Core Concepts

```
Workflow (.github/workflows/ci.yml)
├── Event trigger (on: push, pull_request)
├── Job: build
│   ├── runs-on: ubuntu-latest
│   ├── Step: Checkout code
│   ├── Step: Setup Node.js
│   ├── Step: Install dependencies
│   ├── Step: Run tests
│   └── Step: Build
└── Job: deploy (needs: build)
    ├── runs-on: ubuntu-latest
    └── Step: Deploy to production
```

## Your First Workflow

```yaml
# .github/workflows/ci.yml
name: CI Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build-and-test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run linter
        run: npm run lint

      - name: Run tests
        run: npm test

      - name: Build
        run: npm run build
```

## Event Triggers

```yaml
on:
  # Push to specific branches
  push:
    branches: [main, develop]
    paths:
      - 'src/**'           # Only when source files change

  # Pull requests
  pull_request:
    types: [opened, synchronize]

  # Scheduled (cron)
  schedule:
    - cron: '0 9 * * 1'   # Every Monday at 9 AM

  # Manual trigger
  workflow_dispatch:
    inputs:
      environment:
        type: choice
        options: [staging, production]
```

## Using Actions from Marketplace

The Actions Marketplace has thousands of reusable actions:

```yaml
steps:
  - uses: actions/checkout@v4          # Clone repo
  - uses: actions/setup-node@v4        # Setup Node.js
  - uses: actions/cache@v4             # Cache dependencies
  - uses: docker/build-push-action@v5  # Build Docker images
```

## Job Dependencies

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - run: npm test

  deploy:
    needs: test              # Only runs after "test" succeeds
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - run: echo "Deploying..."
```

## Summary

- GitHub Actions provides CI/CD built directly into GitHub
- Workflows are YAML files in `.github/workflows/`
- Workflows are triggered by events (push, PR, schedule, manual)
- Steps can run commands or use community actions
- Jobs run in parallel by default; use `needs` for dependencies
- Free for public repositories, generous free tier for private
