<Callout type="story" title="The Story">
Imagine you're a new engineer at a startup. Your first task? Deploy a web application. You SSH into the server and see a blinking cursor. No graphical interface. No Start menu. Just a terminal prompt waiting for your command.

This moment — when you face the command line for the first time — is where every DevOps engineer's journey begins. And by the end of this lesson, you'll understand exactly what's running behind that blinking cursor.
</Callout>

## What Is Linux?

Linux is an **open-source operating system kernel** created by Linus Torvalds in 1991. But when people say "Linux," they usually mean a complete operating system — a **Linux distribution** — that bundles the kernel with tools, libraries, and a package manager.

Think of it this way:

- **The Linux kernel** is the engine of a car — it manages hardware, memory, and processes
- **A Linux distribution** is the complete car — engine plus body, seats, dashboard, and everything else you need to drive

<Callout type="info" title="Fun Fact">
Linux runs on everything from your Android phone to the International Space Station. It powers 96.3% of the world's top 1 million servers, all of the top 500 supercomputers, and most of the cloud infrastructure you use every day.
</Callout>

## Why Linux for DevOps?

If you're pursuing a career in DevOps, Cloud, or SRE, Linux isn't optional — it's the foundation everything else is built on:

1. **Servers run Linux** — AWS, GCP, and Azure all default to Linux instances
2. **Containers are Linux** — Docker containers share the host Linux kernel
3. **Kubernetes runs on Linux** — The entire orchestration layer assumes Linux
4. **CI/CD pipelines run on Linux** — GitHub Actions, Jenkins agents, all Linux
5. **Infrastructure tools are Linux-first** — Terraform, Ansible, most DevOps tools

## Linux Distributions

A **distribution** (or "distro") packages the Linux kernel with different software, package managers, and philosophies:

| Distribution | Package Manager | Best For |
|---|---|---|
| **Ubuntu** | apt | Beginners, cloud servers, development |
| **Debian** | apt | Stability, servers |
| **CentOS / Rocky Linux** | dnf/yum | Enterprise, RHEL compatibility |
| **Fedora** | dnf | Latest features, developers |
| **Alpine** | apk | Docker containers (tiny footprint) |
| **Arch Linux** | pacman | Advanced users, customization |

<Callout type="tip" title="Which Distro Should I Use?">
For learning DevOps, start with **Ubuntu**. It has the largest community, the most tutorials, and is the most common choice for cloud servers. Once you're comfortable, you'll naturally explore others.
</Callout>

## Setting Up Your Linux Environment

You have several options to start learning Linux right now:

<TabGroup>
<Tab label="WSL (Windows)">

If you're on Windows, **Windows Subsystem for Linux (WSL)** is the fastest way to get started:

```bash
# Open PowerShell as Administrator and run:
wsl --install

# This installs Ubuntu by default. After restart:
wsl
```

You'll have a full Ubuntu terminal running inside Windows. No virtual machine needed.

</Tab>
<Tab label="macOS">

macOS is Unix-based, so your Terminal app already works similarly to Linux. For a true Linux environment:

```bash
# Option 1: Use the built-in Terminal (most commands work)
# Open: Applications > Utilities > Terminal

# Option 2: Install Lima for a Linux VM
brew install lima
limactl start
```

</Tab>

## Your First Commands

Once you have a terminal open, try these commands:

```bash
# Who am I?
whoami

# Where am I?
pwd

# What's in this directory?
ls

# What operating system is this?
cat /etc/os-release

# What kernel version?
uname -r
```

<Callout type="warning">
Don't worry if you don't understand every command yet. The goal right now is just to get comfortable with the terminal. We'll cover each command in detail in the next lessons.
</Callout>

<Exercise number={2} title="Explore the File System">

Run these commands to explore the Linux file system:

```bash
# List all files including hidden ones
ls -la /home

# See how much disk space is used
df -h

# See how much memory is available
free -h

# See running processes
ps aux | head -10
```

What do you notice about the output of `ls -la`? Can you identify the file permissions, owner, group, and file size columns?

## The Linux Philosophy

Linux was built on a powerful philosophy that shapes how everything in DevOps works:

1. **Everything is a file** — Devices, processes, even network sockets are represented as files
2. **Small, focused tools** — Each program does one thing well
3. **Combine tools with pipes** — Chain small tools together to solve complex problems
4. **Text is the universal interface** — Configuration files are plain text, logs are plain text, everything is human-readable

This philosophy is why Linux is perfect for automation. When everything is a file and every tool accepts text input and produces text output, you can script anything.

<MiniProject title="Your Linux Environment Report">

Create a simple script that generates a report about your Linux environment. You don't need to know scripting yet — just run these commands and save the output:

```bash
echo "=== System Report ==="
echo "Date: $(date)"
echo "Hostname: $(hostname)"
echo "User: $(whoami)"
echo "OS: $(cat /etc/os-release | grep PRETTY_NAME | cut -d'"' -f2)"
echo "Kernel: $(uname -r)"
echo "Uptime: $(uptime -p)"
echo "Disk Usage:"
df -h /
echo "Memory:"
free -h
```

Copy and paste this into your terminal. You've just run your first multi-line command!

</MiniProject>

## Key Takeaways

- Linux is an open-source operating system kernel that powers the vast majority of servers, containers, and cloud infrastructure
- A Linux distribution bundles the kernel with tools and a package manager — Ubuntu is the best choice for beginners
- You can start learning Linux today using WSL (Windows), Terminal (macOS), or a cloud instance
- The Linux philosophy of small, composable tools and text-based configuration is the foundation of DevOps automation
