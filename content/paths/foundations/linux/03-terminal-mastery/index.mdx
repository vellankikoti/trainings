## First, Let's Talk About Your Shell

When you open a terminal, you're talking to a program called a **shell**. The shell reads what you type, figures out what you mean, and tells the operating system to do it.

The most common shell is **Bash** (Bourne Again SHell). It's the default on most Linux systems, and it's what we'll use throughout this course.

```bash
# Which shell are you running right now?
echo $SHELL

# What version?
bash --version | head -1
```

Think of Bash as your translator. You speak human. The kernel speaks machine. Bash sits between you and translates in both directions.

## The Shortcuts That Change Everything

I'm going to teach you a bunch of shortcuts. But here's the truth — you only need to memorize three to start. Everything else is bonus. Here are your Big Three:

### 1. `Ctrl + R` — Search Your History

This is the single most powerful shortcut in bash. Press `Ctrl + R` and start typing any part of a previous command. Bash will find it for you.

```bash
# Let's say earlier you typed:
ssh -i ~/keys/production.pem ubuntu@10.0.1.42

# Now you need to run it again.
# Instead of retyping all of that:

# Press Ctrl + R
# Type: ssh
# Bash instantly finds: ssh -i ~/keys/production.pem ubuntu@10.0.1.42
# Press Enter to run it

# If it finds the wrong match, press Ctrl + R again to cycle through older matches
# Press Ctrl + G to cancel and go back to a blank prompt
```

Let's practice this right now:

```bash
# Run a few commands to build up some history:
echo "This is my first command"
ls -la /etc
cat /etc/hostname
echo "DevOps is going to be great"

# Now press Ctrl + R and type "DevOps"
# Watch it find your echo command instantly
```

<Callout type="tip" title="This alone saves hours">
Once you build the `Ctrl + R` habit, you'll stop retyping anything. That `kubectl` command you ran 45 minutes ago? `Ctrl + R`, type a few letters, done. That complex `docker` command from yesterday? Same thing. Your history is a personal command library that grows every day.
</Callout>

### 2. `Tab` — Autocomplete Everything

Press `Tab` once to autocomplete a command, file name, or path. Press it twice to see all possible completions.

```bash
# Type "system" then press Tab twice:
system⇥⇥
# Shows: systemctl  systemd-analyze  systemd-cat  ...

# Type "systemctl sta" then press Tab:
systemctl sta⇥
# Completes to: systemctl status

# Type "ls /etc/ho" then press Tab:
ls /etc/ho⇥
# Completes to: ls /etc/hostname (or shows options if there are multiple matches)
```

Here's the rule: **If you're typing a full path or command name, you're working too hard.** Tab does most of it for you.

```bash
# Instead of typing this:
cat /etc/systemd/system/multi-user.target.wants/

# Type this:
cat /etc/sys⇥/sy⇥/mu⇥.ta⇥.wa⇥/
# Tab fills in each piece as you go
```

### 3. `Ctrl + A` and `Ctrl + E` — Jump to Start and End

You're editing a long command and realize you need to change something at the beginning:

```bash
# You've typed this, but want to add "sudo" at the start:
apt install nginx
# Don't reach for the Home key or use arrow keys. Press:
# Ctrl + A → cursor jumps to the beginning
# Type: sudo
# Ctrl + E → cursor jumps back to the end
```

That's it. Those are your Big Three. Master these three shortcuts and you'll already be faster than 80% of terminal users.

Let's build muscle memory. Do each of these steps IN ORDER — don't skip:

**Round 1: History Search**

```bash

```

How did that feel? Clunky? That's normal. Do it 10 more times today and it'll start feeling natural. By next week, you won't even think about it.

## The Full Shortcut Arsenal

The Big Three will carry you far. But when you're ready for more, here's the complete toolkit:

### Moving Around

| Shortcut | What It Does | When You'll Use It |
|---|---|---|
| `Ctrl + A` | Jump to beginning of line | Add `sudo` to a command you already typed |
| `Ctrl + E` | Jump to end of line | After editing the middle of a command |
| `Alt + F` | Jump forward one word | Navigate through long commands |
| `Alt + B` | Jump backward one word | Same — word by word is faster than character by character |
| `Ctrl + L` | Clear the screen | When your terminal is cluttered (way better than typing `clear`) |

### Editing

| Shortcut | What It Does | When You'll Use It |
|---|---|---|
| `Ctrl + U` | Delete from cursor to beginning | Clear a half-typed command and start over |
| `Ctrl + K` | Delete from cursor to end | Remove the end of a command |
| `Ctrl + W` | Delete the previous word | Fix a typo in the last word you typed |
| `Alt + D` | Delete the next word | Remove a word ahead of the cursor |
| `Ctrl + Y` | Paste what you just deleted | Undo your deletion — Bash remembers what you cut |

### Control

| Shortcut | What It Does | When You'll Use It |
|---|---|---|
| `Ctrl + C` | Kill the running process | Something is running forever — stop it now |
| `Ctrl + D` | Exit the shell (or send EOF) | When you want to close the terminal or end input |
| `Ctrl + Z` | Suspend the running process | Pause something to do something else, then come back |

<Callout type="info" title="About Ctrl+C vs Ctrl+Z">
`Ctrl + C` **kills** the process. It's gone. Dead. `Ctrl + Z` **pauses** it. It's still alive, just sleeping in the background. You can bring it back with `fg` (foreground). This matters when you're editing a file in vim and need to quickly check something — `Ctrl + Z` to pause vim, do your thing, then `fg` to come back.
</Callout>

## Command History — Your Personal Command Library

Every command you've ever typed is saved. This history is pure gold once you learn how to use it.

```bash
# See your recent commands:
history | tail -20

# How many commands have you run total?
history | wc -l
```

### History Shortcuts (Beyond Ctrl+R)

```bash
# Re-run the last command:
!!

# This is incredibly useful with sudo:
apt install nginx
# "Permission denied"
sudo !!
# Runs: sudo apt install nginx

# Re-run command #42 from history:
!42

# Re-run the last command that started with "ssh":
!ssh
# Careful with this one — make sure you remember what that command was
```

### Reusing Pieces of Previous Commands

This is where it gets really clever:

```bash
# Grab the LAST ARGUMENT of the previous command:
mkdir -p /opt/myapp/config
cd !$
# cd /opt/myapp/config — !$ means "last argument"

# Grab ALL arguments of the previous command:
echo one two three
echo !*
# echo one two three — !* means "all arguments"

# Fix a typo in the last command:
cat /etc/hotsname
^hotsname^hostname
# Runs: cat /etc/hostname — substitutes the typo
```

<Exercise number={2} title="History Power Moves">

Let's practice these history features:

```bash
# Step 1: Run some commands
mkdir -p /tmp/history-practice/deep/nested/path
echo "History is powerful"
ls -la /var/log

# Step 2: Use !! (repeat last command)
sudo !!
# This should run: sudo ls -la /var/log

# Step 3: Use !$ (last argument)
cd !$
# You should now be in /var/log
pwd
# Verify you're in /var/log

# Step 4: Go back and use history search
cd ~
# Press Ctrl+R, type "mkdir"
# It should find your mkdir command from Step 1
# Press Enter to run it again (it's safe — mkdir -p doesn't fail if it exists)

# Step 5: Check your history
history | tail -10
# See all the commands you just ran, numbered and timestamped
```

## Tab Completion and Globbing — Type Less, Do More

You already know Tab completes things. Let's go deeper.

### Install Better Completion

```bash
# Install enhanced bash completion (more commands get Tab support):
sudo apt install -y bash-completion

# Reload your shell:
source ~/.bashrc
```

Now Tab works with more commands — try `systemctl status ⇥⇥` and watch it list all services.

### Globbing — Wildcards That Match Patterns

Bash can expand special characters BEFORE running a command. This is called globbing:

| Pattern | Matches | Example |
|---|---|---|
| `*` | Anything (any number of characters) | `ls *.log` — all files ending in .log |
| `?` | Exactly one character | `ls file?.txt` — file1.txt, file2.txt, not file10.txt |
| `[abc]` | Any one of these characters | `ls file[123].txt` — file1.txt, file2.txt, file3.txt |
| `[0-9]` | Any digit | `ls log.[0-9]` — log.1, log.2, etc. |
| `{a,b,c}` | Each of these literally | `mkdir {src,tests,docs}` — creates three directories |

```bash
# Real-world examples:

# List all config files:
ls /etc/*.conf

# Create an entire project structure in one command:
mkdir -p project/{src,tests,docs,scripts,config}

# List all log files from today:
ls /var/log/*.log

# Find all files starting with "host":
ls /etc/host*
```

<Exercise number={3} title="Globbing Practice">

```bash
# Set up a practice area:
mkdir -p ~/devops-learning/glob-practice
cd ~/devops-learning/glob-practice

# Create a bunch of files:
touch app.log error.log debug.log access.log
touch config.yml config.json config.toml
touch server1.cfg server2.cfg server3.cfg
touch notes.txt readme.txt changelog.txt

# Now use globbing to work with groups of files:

# 1. List only the log files:
ls *.log

# 2. List only the config files (any extension):
ls config.*

# 3. List server configs with single-digit numbers:
ls server?.cfg

# 4. List only .txt files:
ls *.txt

# 5. Count how many .log files exist:
ls *.log | wc -l

# 6. Create backup copies of all configs in one command:
mkdir backups
cp config.* backups/

# 7. Verify the backups:
ls backups/

# Clean up:
cd ~/devops-learning
rm -rf glob-practice
```

Notice how you never had to list files one by one? Globbing lets you work with patterns instead of individual names. On a real server with thousands of files, this is essential.

</Exercise>

## Make the Terminal Yours — Aliases and Functions

Here's where it starts feeling like YOUR terminal instead of a generic one. Aliases let you create shortcuts for commands you use all the time.

```bash
# Instead of typing "ls -la" every time:
alias ll='ls -la'

# Instead of typing "cd ..":
alias ..='cd ..'
alias ...='cd ../..'

# DevOps shortcuts you'll use daily:
alias k='kubectl'           # You'll thank me when we get to Kubernetes
alias d='docker'             # Same for Docker
alias g='git'                # And Git
alias tf='terraform'         # And Terraform
alias gs='git status'        # Most-used git command
alias gd='git diff'
```

**Functions** are aliases on steroids — they can take arguments:

```bash
# Create a directory and immediately cd into it:
mkcd() { mkdir -p "$1" && cd "$1"; }

# Usage:
mkcd ~/projects/my-new-app
# Creates the directory AND moves into it. One command.

# Quick port check — is something running on a port?
port() { ss -tlnp | grep ":$1 "; }

# Usage:
port 80
# Shows whatever is listening on port 80
```

```bash
# Now add your custom configuration:
cat >> ~/.bashrc << 'MYCONFIG'

# ====================================
# My DevOps Terminal Configuration

# --- Navigation Shortcuts ---
alias ..='cd ..'
alias ...='cd ../..'
alias ....='cd ../../..'
alias ll='ls -la --color=auto'

alias l='ls -CF'

# --- Quick System Info ---
alias ports='ss -tlnp'
alias mem='free -h'
alias disk='df -h /'
alias cpu='top -bn1 | head -5'
alias myip='curl -s ifconfig.me'

# --- Safety Nets ---
alias rm='rm -i'          # Ask before deleting
alias cp='cp -i'          # Ask before overwriting
alias mv='mv -i'          # Ask before overwriting

# --- DevOps Tools (we'll use these later) ---
alias k='kubectl'
alias d='docker'
alias g='git'
alias tf='terraform'
alias gs='git status'
alias gd='git diff'
alias gl='git log --oneline -10'

# --- Custom Functions ---
mkcd() { mkdir -p "$1" && cd "$1"; }
port() { ss -tlnp 2>/dev/null | grep ":$1 " || lsof -i ":$1" 2>/dev/null; }
extract() {
  case "$1" in
    *.tar.gz)  tar xzf "$1" ;;
    *.tar.bz2) tar xjf "$1" ;;
    *.tar.xz)  tar xf "$1" ;;
    *.zip)     unzip "$1" ;;
    *.gz)      gunzip "$1" ;;
    *) echo "Don't know how to extract: $1" ;;
  esac
}

# --- Better History ---
export HISTSIZE=10000            # Remember more commands
export HISTFILESIZE=20000        # Store more in the file
export HISTCONTROL=ignoredups:erasedups  # No duplicate entries
shopt -s histappend              # Append to history, don't overwrite

# --- Colored Prompt ---
export PS1='\[\e[32m\]\u\[\e[0m\]:\[\e[34m\]\w\[\e[0m\]\$ '

MYCONFIG

# Reload the configuration:
source ~/.bashrc

# Test it:
echo ""
echo "Testing your new configuration:"
echo "================================"
ll /etc | head -5
echo ""
echo "Your aliases:"
alias | head -10
echo ""
echo "Your IP: $(myip 2>/dev/null || echo 'could not reach internet')"
echo ""
echo "Configuration loaded successfully."
```

Try out your new shortcuts:

```bash

..             # Goes up one directory
ll             # Detailed file listing

# System info:
disk           # Shows disk usage
mem            # Shows memory usage

# Functions:
mkcd ~/devops-learning/test-dir   # Creates and enters directory
pwd            # Verify you're in the new directory

# Clean up test:
cd ~/devops-learning
rmdir test-dir
```

This configuration will follow you through the rest of this course and into your career. As you learn new tools, you'll add new aliases. Your `.bashrc` is a living document that grows with you.
