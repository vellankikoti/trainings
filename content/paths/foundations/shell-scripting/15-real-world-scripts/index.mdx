## Log Rotation Script

Log files grow indefinitely if left unchecked. A log rotator compresses old logs, shifts numbered archives, and removes files past a retention period. This is what `logrotate` does under the hood, but understanding how to build one teaches you file manipulation, compression, and atomic operations:

```bash
#!/bin/bash
set -euo pipefail

# Simple log rotation: rotate when a file exceeds a size threshold
# Usage: rotate_log /path/to/file.log [max_size_kb] [keep_count]

rotate_log() {
    local log_file="$1"
    local max_size_kb="${2:-1024}"    # Default: 1 MB
    local keep_count="${3:-5}"         # Default: keep 5 rotated copies

    if [ ! -s "$log_file" ]; then
        return 0
    fi

    local current_size
    current_size=$(du -k "$log_file" | cut -f1)

    if [ "$current_size" -lt "$max_size_kb" ]; then
        echo "  $log_file: ${current_size}KB < ${max_size_kb}KB — no rotation needed"
        return 0
    fi

    echo "  $log_file: ${current_size}KB >= ${max_size_kb}KB — rotating"

    # Shift existing rotated files: .4.gz -> .5.gz, .3.gz -> .4.gz, etc.
    for ((i = keep_count; i > 1; i--)); do
        local prev=$((i - 1))
        [ -f "${log_file}.${prev}.gz" ] && mv "${log_file}.${prev}.gz" "${log_file}.${i}.gz"
    done

    # Compress the current log to .1.gz
    gzip -c "$log_file" > "${log_file}.1.gz"

    # Truncate the original file — keeps the inode so processes can keep writing
    : > "$log_file"
    echo "  Compressed to ${log_file}.1.gz, truncated original"

    # Remove files beyond the retention count
    for ((i = keep_count + 1; i <= keep_count + 5; i++)); do
        [ -f "${log_file}.${i}.gz" ] && rm -f "${log_file}.${i}.gz"
    done
}
```

<Callout type="tip" title="Truncate vs Remove">
Never `rm` and recreate an active log file. Processes that have the file open will keep writing to the deleted file (the inode persists until all file descriptors close). Instead, truncate with `: > file` or `truncate -s 0 file`. This empties the file without changing the inode, so running processes continue writing to it seamlessly.
</Callout>

### Configuration-Driven Rotation

A production rotator reads a config file that specifies which logs to rotate and how:

```bash
#!/bin/bash
set -euo pipefail

# Config file format: path  max_size_kb  retain_count
# Lines starting with # are comments

process_rotation_config() {
    local config_file="$1"
    local rotated=0
    local skipped=0

    echo "=== Log Rotation Run ==="
    echo "Config: $config_file"
    echo ""

    while IFS= read -r line; do
        # Skip comments and empty lines
        [[ "$line" =~ ^[[:space:]]*# ]] && continue
        [[ -z "${line// /}" ]] && continue

        read -r path max_size retain <<< "$line"

        if [ -f "$path" ]; then
            rotate_log "$path" "$max_size" "$retain"
            rotated=$((rotated + 1))
        else
            echo "  SKIP: $path (not found)"
            skipped=$((skipped + 1))
        fi
    done < "$config_file"

    echo ""
    echo "Rotation complete: $rotated processed, $skipped skipped"
}
```

## Backup System

A reliable backup script needs incremental support, retention policies, verification, and failure notification. The `rsync --link-dest` trick is the key — unchanged files become hard links to the previous backup, consuming no extra disk space:

```bash
#!/bin/bash
set -euo pipefail

BACKUP_BASE="/backups"
TIMESTAMP=$(date '+%Y%m%d_%H%M%S')
RETENTION_DAYS=30

log_backup() {
    local level="$1"; shift
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [$level] $*"
}

# Full backup: complete copy
do_full_backup() {
    local source="$1"
    local dest="$BACKUP_BASE/full_$TIMESTAMP"

    log_backup INFO "Starting full backup: $source -> $dest"
    mkdir -p "$dest"

    if rsync -a --stats "$source/" "$dest/" 2>&1; then
        log_backup INFO "Full backup complete"
        echo "$dest" > "$BACKUP_BASE/.latest_full"
        return 0
    else
        log_backup ERROR "Full backup FAILED"
        return 1
    fi
}

# Incremental backup: only changed files since last full
do_incremental_backup() {
    local source="$1"

    if [ ! -f "$BACKUP_BASE/.latest_full" ]; then
        log_backup WARN "No full backup found — falling back to full"
        do_full_backup "$source"
        return $?
    fi

    local reference
    reference=$(cat "$BACKUP_BASE/.latest_full")
    local dest="$BACKUP_BASE/incr_$TIMESTAMP"

    log_backup INFO "Starting incremental: reference=$reference"
    mkdir -p "$dest"

    # --link-dest creates hard links for unchanged files
    rsync -a --stats --link-dest="$reference" "$source/" "$dest/" 2>&1
}

# Verify backup integrity
verify_backup() {
    local backup_dir="$1"
    local file_count
    file_count=$(find "$backup_dir" -type f | wc -l)

    if [ "$file_count" -eq 0 ]; then
        log_backup ERROR "Backup is empty: $backup_dir"
        return 1
    fi

    log_backup INFO "Verified: $file_count files in $backup_dir"
}

# Remove old backups
apply_retention() {
    log_backup INFO "Applying retention: removing backups older than $RETENTION_DAYS days"
    find "$BACKUP_BASE" -maxdepth 1 \( -name "full_*" -o -name "incr_*" \) -type d \
        -mtime +"$RETENTION_DAYS" -exec rm -rf {} \;
}
```

<Callout type="warning" title="Backup Verification Is Not Optional">
A backup that has never been verified is not a backup — it is a hope. Always verify after writing: check file counts, test restoration of a sample file, or compute checksums. Many teams have discovered their "backups" were empty or corrupted only when they needed to restore.
</Callout>

## Deployment Script

A deployment script pulls code, builds, runs health checks, and rolls back on failure. The symlink swap pattern is how most production deployments work:

```bash
#!/bin/bash
set -euo pipefail

DEPLOY_DIR="/opt/webapp"
RELEASES_DIR="$DEPLOY_DIR/releases"
CURRENT_LINK="$DEPLOY_DIR/current"
KEEP_RELEASES=5
HEALTH_TIMEOUT=30

log_deploy() { echo "[$(date '+%H:%M:%S')] [deploy] $*"; }

# Atomic symlink switch — zero-downtime deployment
activate_release() {
    local release_dir="$1"

    # Save previous release for rollback
    if [ -L "$CURRENT_LINK" ]; then
        readlink -f "$CURRENT_LINK" > "$DEPLOY_DIR/.previous_release"
    fi

    # Atomic switch: ln + mv is atomic on the same filesystem
    local tmp_link="${CURRENT_LINK}.tmp"
    ln -sfn "$release_dir" "$tmp_link"
    mv -Tf "$tmp_link" "$CURRENT_LINK"

    log_deploy "Activated: $release_dir"
}

# Health check with retry loop
health_check() {
    local url="$1"
    local timeout="$2"
    local elapsed=0

    log_deploy "Health checking: $url (timeout: ${timeout}s)"
    while [ "$elapsed" -lt "$timeout" ]; do
        if curl -sf "$url" > /dev/null 2>&1; then
            log_deploy "  Health check PASSED (${elapsed}s)"
            return 0
        fi
        sleep 2
        elapsed=$((elapsed + 2))
    done

    log_deploy "  Health check FAILED after ${timeout}s"
    return 1
}

# Rollback to previous release
rollback() {
    if [ ! -f "$DEPLOY_DIR/.previous_release" ]; then
        log_deploy "No previous release — cannot rollback!"
        return 1
    fi

    local previous
    previous=$(cat "$DEPLOY_DIR/.previous_release")
    activate_release "$previous"
    log_deploy "Rolled back to: $previous"
}
```

<Callout type="tip" title="Atomic Symlink Switch">
A simple `ln -sf` is not atomic — it first removes the old link, then creates a new one. During that brief moment, the link does not exist. Use `ln -sfn new tmp && mv -Tf tmp current` instead. The `mv` (rename) is atomic on the same filesystem, guaranteeing the symlink always points somewhere valid.
</Callout>

### Release Cleanup

Keep only the N most recent releases to prevent disk exhaustion:

```bash
cleanup_old_releases() {
    local keep="$1"
    local release_count
    release_count=$(ls -1 "$RELEASES_DIR" 2>/dev/null | wc -l)

    if [ "$release_count" -gt "$keep" ]; then
        ls -1t "$RELEASES_DIR" | tail -n +$((keep + 1)) | while read -r old; do
            log_deploy "Removing old release: $old"
            rm -rf "${RELEASES_DIR:?}/$old"
        done
    fi
}
```

## Monitoring Script

A monitoring script checks system resources, compares against thresholds, and raises alerts. The key features are cooldown (to prevent alert storms) and graceful threshold handling:

```bash
#!/bin/bash
set -euo pipefail

# Thresholds
CPU_WARN=75   CPU_CRIT=90
MEM_WARN=80   MEM_CRIT=95
DISK_WARN=80  DISK_CRIT=90

COOLDOWN_FILE="/tmp/monitor-cooldown"
COOLDOWN_SECONDS=300

check_cpu() {
    local idle total idle2 total2
    read -r _ _ _ _ idle _ <<< "$(head -1 /proc/stat)"
    read -r _ total <<< "$(awk '/^cpu / {s=0; for(i=2;i<=NF;i++) s+=$i; print $1, s}' /proc/stat)"
    sleep 1
    read -r _ _ _ _ idle2 _ <<< "$(head -1 /proc/stat)"
    read -r _ total2 <<< "$(awk '/^cpu / {s=0; for(i=2;i<=NF;i++) s+=$i; print $1, s}' /proc/stat)"
    echo $((100 - ((idle2 - idle) * 100 / (total2 - total + 1))))
}

check_memory() {
    local total used
    read -r total used <<< "$(free | awk '/^Mem:/ {print $2, $3}')"
    echo $((used * 100 / (total + 1)))
}

check_disk() {
    df "${1:-/}" | awk 'NR==2 {gsub(/%/,""); print $5}'
}

# Alert with cooldown to prevent notification storms
alert() {
    local check_name="$1" level="$2" message="$3"

    if [ -f "$COOLDOWN_FILE" ]; then
        local last
        last=$(grep "^${check_name}:" "$COOLDOWN_FILE" 2>/dev/null | cut -d: -f2 || echo "0")
        [ -n "$last" ] && [ $(($(date +%s) - last)) -lt "$COOLDOWN_SECONDS" ] && return 0
    fi

    echo "  [$level] $message"
    # In production: curl webhook, mail, PagerDuty API call, etc.
    grep -v "^${check_name}:" "$COOLDOWN_FILE" > "${COOLDOWN_FILE}.tmp" 2>/dev/null || true
    mv "${COOLDOWN_FILE}.tmp" "$COOLDOWN_FILE" 2>/dev/null || true
    echo "${check_name}:$(date +%s)" >> "$COOLDOWN_FILE"
}

evaluate() {
    local name="$1" value="$2" warn="$3" crit="$4"
    if [ "$value" -ge "$crit" ]; then
        alert "$name" "CRITICAL" "$name at ${value}% (threshold: ${crit}%)"
    elif [ "$value" -ge "$warn" ]; then
        alert "$name" "WARNING" "$name at ${value}% (threshold: ${warn}%)"
    else
        printf "  [OK]       %-10s %d%%\n" "$name" "$value"
    fi
}
```

### Service Health Checks

Beyond system resources, check that services are actually responding:

```bash
# Check TCP port connectivity
check_service() {
    local name="$1" host="$2" port="$3" timeout="${4:-3}"
    if timeout "$timeout" bash -c "echo >/dev/tcp/$host/$port" 2>/dev/null; then
        printf "  [UP]   %-20s %s:%s\n" "$name" "$host" "$port"
        return 0
    else
        printf "  [DOWN] %-20s %s:%s\n" "$name" "$host" "$port"
        return 1
    fi
}

# Check HTTP endpoint
check_http() {
    local name="$1" url="$2" expected="${3:-200}"
    local code
    code=$(curl -s -o /dev/null -w "%{http_code}" --connect-timeout 5 "$url" 2>/dev/null || echo "000")
    if [ "$code" = "$expected" ]; then
        printf "  [OK]   %-20s HTTP %s\n" "$name" "$code"
    else
        printf "  [FAIL] %-20s HTTP %s (expected %s)\n" "$name" "$code" "$expected"
        return 1
    fi
}

# Check process is running
check_process() {
    local name="$1" proc_name="$2"
    if pgrep -x "$proc_name" > /dev/null 2>&1; then
        printf "  [RUNNING] %-16s PID %s\n" "$name" "$(pgrep -x "$proc_name" | head -1)"
    else
        printf "  [STOPPED] %-16s\n" "$name"
        return 1
    fi
}
```

## The Script Structure Pattern

Every production-grade script follows the same structure. Once you internalize it, writing new scripts becomes filling in the blanks:

```bash
#!/bin/bash
set -euo pipefail

# 1. Configuration (defaults that can be overridden)
APP_NAME="myservice"
LOG_FILE="/var/log/${APP_NAME}.log"
LOCK_FILE="/tmp/${APP_NAME}.lock"

# 2. Utility functions (logging, cleanup, notification)
log() { echo "[$(date '+%H:%M:%S')] $*"; }
cleanup() { rm -f "$LOCK_FILE"; }
trap cleanup EXIT

# 3. Core functions (the actual business logic)
do_task_a() { ... }
do_task_b() { ... }
verify()    { ... }

# 4. Main function (orchestrates everything)
main() {
    log "Starting $APP_NAME"

    exec 200>"$LOCK_FILE"
    flock -n 200 || { log "Already running"; exit 0; }

    do_task_a
    do_task_b
    verify

    log "Complete"
}

# 5. Entry point
main "$@"
```

```bash
cat > ~/log-rotator.sh << 'SCRIPT'
#!/bin/bash
set -euo pipefail

# Complete log rotation system with compression and retention

DEMO_DIR="/tmp/log-rotation-demo"
LOG_DIR="$DEMO_DIR/logs"
CONFIG_FILE="$DEMO_DIR/rotation.conf"
STATE_FILE="$DEMO_DIR/.rotation-state"
ROTATION_LOG="$DEMO_DIR/rotation.log"

cleanup() { rm -rf "$DEMO_DIR"; }
trap cleanup EXIT

mkdir -p "$LOG_DIR"

# --- Utility functions ---

log() {
    local msg="[$(date '+%Y-%m-%d %H:%M:%S')] $*"
    echo "$msg"
    echo "$msg" >> "$ROTATION_LOG"
}

human_size() {
    local bytes="$1"
    if [ "$bytes" -ge 1048576 ]; then
        echo "$((bytes / 1048576)) MB"
    elif [ "$bytes" -ge 1024 ]; then
        echo "$((bytes / 1024)) KB"
    else
        echo "$bytes bytes"
    fi
}

# --- Create sample log files of various sizes ---

generate_sample_logs() {
    log "Generating sample log files..."

    # Large access log (should be rotated)
    for i in $(seq 1 200); do
        echo "[$(date '+%Y-%m-%d %H:%M:%S')] INFO  Request $i from 10.0.$((RANDOM % 255)).$((RANDOM % 255)) — ${RANDOM}ms"
    done > "$LOG_DIR/access.log"

    # Medium error log (should be rotated with lower threshold)
    for i in $(seq 1 80); do
        echo "[$(date '+%Y-%m-%d %H:%M:%S')] ERROR db-0$((RANDOM % 3 + 1)) connection timeout after 30s"
    done > "$LOG_DIR/error.log"

    # Small debug log (should NOT be rotated)
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] INFO  Application started" > "$LOG_DIR/debug.log"

    # Simulate already-rotated archives from previous runs
    echo "old rotated content round 1" | gzip > "$LOG_DIR/access.log.1.gz"
    echo "old rotated content round 2" | gzip > "$LOG_DIR/access.log.2.gz"
    echo "old rotated content round 3" | gzip > "$LOG_DIR/access.log.3.gz"

    for f in "$LOG_DIR"/*; do
        local size
        size=$(stat -c%s "$f" 2>/dev/null || echo 0)
        log "  Created: $(basename "$f") ($(human_size "$size"))"
    done
}

# --- Rotation configuration ---

cat > "$CONFIG_FILE" << EOF
# Log Rotation Configuration
# Format: path  max_bytes  retain_count  compress
$LOG_DIR/access.log   5000   4   yes
$LOG_DIR/error.log    2000   3   yes
$LOG_DIR/debug.log    50000  2   yes
EOF

# --- Core rotation function ---

rotate_single_log() {
    local log_file="$1"
    local max_bytes="$2"
    local retain="$3"
    local compress="$4"

    if [ ! -f "$log_file" ]; then
        log "  SKIP: $log_file (not found)"
        return 0
    fi

    local current_size
    current_size=$(stat -c%s "$log_file" 2>/dev/null || echo 0)

    if [ "$current_size" -lt "$max_bytes" ]; then
        log "  OK: $(basename "$log_file") — $(human_size "$current_size") < $(human_size "$max_bytes")"
        return 0
    fi

    log "  ROTATE: $(basename "$log_file") — $(human_size "$current_size") >= $(human_size "$max_bytes")"

    # Remove the oldest retained file
    if [ "$compress" = "yes" ]; then
        rm -f "${log_file}.${retain}.gz"
    else
        rm -f "${log_file}.${retain}"
    fi

    # Shift existing rotated files: .3 -> .4, .2 -> .3, .1 -> .2
    for ((i = retain - 1; i >= 1; i--)); do
        local next=$((i + 1))
        if [ "$compress" = "yes" ]; then
            [ -f "${log_file}.${i}.gz" ] && mv "${log_file}.${i}.gz" "${log_file}.${next}.gz"
        else
            [ -f "${log_file}.${i}" ] && mv "${log_file}.${i}" "${log_file}.${next}"
        fi
    done

    # Compress and rotate the current log to .1
    if [ "$compress" = "yes" ]; then
        gzip -c "$log_file" > "${log_file}.1.gz"
        local compressed_size
        compressed_size=$(stat -c%s "${log_file}.1.gz" 2>/dev/null || echo 0)
        if [ "$current_size" -gt 0 ]; then
            local ratio=$((100 - (compressed_size * 100 / current_size)))
            log "    Compressed: $(human_size "$current_size") -> $(human_size "$compressed_size") (${ratio}% reduction)"
        fi
    else
        cp "$log_file" "${log_file}.1"
    fi

    # Truncate original (preserves inode for processes still writing)
    : > "$log_file"
    log "    Truncated original file"

    # Record rotation event
    echo "$(date '+%Y-%m-%d %H:%M:%S') $(basename "$log_file") $(human_size "$current_size")" >> "$STATE_FILE"
}

# --- Run rotation from config ---

run_rotation() {
    log ""
    log "=========================================="
    log "  Log Rotation Run"
    log "  Config: $CONFIG_FILE"
    log "=========================================="

    while IFS= read -r line; do
        [[ "$line" =~ ^[[:space:]]*# ]] && continue
        [[ -z "${line// /}" ]] && continue
        read -r path max_bytes retain compress <<< "$line"
        rotate_single_log "$path" "$max_bytes" "$retain" "$compress"
    done < "$CONFIG_FILE"

    log ""
    log "Rotation run complete"
}

# --- Execute demo ---

generate_sample_logs
echo ""

echo "=== Before Rotation ==="
ls -lh "$LOG_DIR/"
echo ""

run_rotation

echo ""
echo "=== After Rotation ==="
ls -lh "$LOG_DIR/"

echo ""
echo "=== Rotation History ==="
if [ -f "$STATE_FILE" ]; then
    cat "$STATE_FILE"
else
    echo "No rotations performed"
fi
SCRIPT

chmod +x ~/log-rotator.sh
~/log-rotator.sh
```

<Exercise number={2} title="Health Check Suite">

```bash
cat > ~/health-suite.sh << 'SCRIPT'
#!/bin/bash
set -uo pipefail

# Health Check Suite
# Checks multiple services and system resources, produces a status report

CHECKS_PASSED=0
CHECKS_FAILED=0
CHECKS_WARNED=0
START_TIME=$(date +%s)

# --- Formatting ---

section() {
    echo ""
    echo "=== $1 ==="
}

record() {
    local status="$1" check="$2" detail="$3"
    case "$status" in
        OK)   printf "  [OK]       %-25s %s\n" "$check" "$detail"; CHECKS_PASSED=$((CHECKS_PASSED + 1)) ;;
        WARN) printf "  [WARNING]  %-25s %s\n" "$check" "$detail"; CHECKS_WARNED=$((CHECKS_WARNED + 1)) ;;
        FAIL) printf "  [FAILED]   %-25s %s\n" "$check" "$detail"; CHECKS_FAILED=$((CHECKS_FAILED + 1)) ;;
    esac
}

# --- System Resource Checks ---

check_system_resources() {
    section "System Resources"

    # CPU usage (1-second sample)
    local cpu_idle
    cpu_idle=$(top -bn1 2>/dev/null | awk '/^%Cpu/ {print int($8)}' || echo "95")
    local cpu_used=$((100 - cpu_idle))
    if [ "$cpu_used" -gt 90 ]; then
        record FAIL "CPU Usage" "${cpu_used}% (critical > 90%)"
    elif [ "$cpu_used" -gt 75 ]; then
        record WARN "CPU Usage" "${cpu_used}% (warning > 75%)"
    else
        record OK "CPU Usage" "${cpu_used}%"
    fi

    # Memory usage
    local mem_total mem_used mem_pct
    read -r mem_total mem_used <<< "$(free 2>/dev/null | awk '/^Mem:/ {print $2, $3}' || echo "1 1")"
    mem_pct=$((mem_used * 100 / (mem_total + 1)))
    if [ "$mem_pct" -gt 95 ]; then
        record FAIL "Memory Usage" "${mem_pct}%"
    elif [ "$mem_pct" -gt 80 ]; then
        record WARN "Memory Usage" "${mem_pct}%"
    else
        record OK "Memory Usage" "${mem_pct}%"
    fi

    # Disk usage on root
    local disk_pct
    disk_pct=$(df / | awk 'NR==2 {gsub(/%/,""); print $5}')
    if [ "$disk_pct" -gt 90 ]; then
        record FAIL "Disk Usage (/)" "${disk_pct}%"
    elif [ "$disk_pct" -gt 80 ]; then
        record WARN "Disk Usage (/)" "${disk_pct}%"
    else
        record OK "Disk Usage (/)" "${disk_pct}%"
    fi

    # Load average
    local load_1m
    load_1m=$(awk '{printf "%.0f", $1}' /proc/loadavg 2>/dev/null || echo "0")
    local cpu_count
    cpu_count=$(nproc 2>/dev/null || echo "1")
    if [ "$load_1m" -gt $((cpu_count * 2)) ]; then
        record FAIL "Load Average" "${load_1m} (CPUs: $cpu_count)"
    elif [ "$load_1m" -gt "$cpu_count" ]; then
        record WARN "Load Average" "${load_1m} (CPUs: $cpu_count)"
    else
        record OK "Load Average" "${load_1m} (CPUs: $cpu_count)"
    fi

    # Inode usage
    local inode_pct
    inode_pct=$(df -i / 2>/dev/null | awk 'NR==2 {gsub(/%/,""); print $5}' || echo "0")
    if [ "$inode_pct" -gt 90 ]; then
        record FAIL "Inode Usage (/)" "${inode_pct}%"
    elif [ "$inode_pct" -gt 80 ]; then
        record WARN "Inode Usage (/)" "${inode_pct}%"
    else
        record OK "Inode Usage (/)" "${inode_pct}%"
    fi
}

# --- Network Connectivity Checks ---

check_connectivity() {
    section "Network Connectivity"

    # DNS resolution
    if getent hosts google.com &>/dev/null || host google.com &>/dev/null 2>&1; then
        record OK "DNS Resolution" "google.com resolves"
    else
        record FAIL "DNS Resolution" "Cannot resolve google.com"
    fi

    # Port checks
    local services=("22:SSH" "80:HTTP" "443:HTTPS" "5432:PostgreSQL" "6379:Redis")
    for entry in "${services[@]}"; do
        local port="${entry%%:*}"
        local name="${entry#*:}"
        if timeout 2 bash -c "echo >/dev/tcp/localhost/$port" 2>/dev/null; then
            record OK "$name (port $port)" "Listening"
        else
            record WARN "$name (port $port)" "Not listening"
        fi
    done
}

# --- Process Checks ---

check_processes() {
    section "Key Processes"

    local procs=("bash" "init" "sshd" "cron" "nginx" "postgres")
    for proc in "${procs[@]}"; do
        if pgrep -x "$proc" &>/dev/null; then
            local count
            count=$(pgrep -x "$proc" | wc -l)
            record OK "$proc" "$count instance(s)"
        else
            record WARN "$proc" "Not running"
        fi
    done
}

# --- Filesystem Checks ---

check_filesystem() {
    section "Filesystem Health"

    # Writable directories
    for dir in /tmp /var/log; do
        if [ -w "$dir" ]; then
            record OK "Writable: $dir" "OK"
        else
            record FAIL "Writable: $dir" "Not writable"
        fi
    done

    # Large files in /tmp
    local large_count
    large_count=$(find /tmp -maxdepth 1 -type f -size +100M 2>/dev/null | wc -l)
    if [ "$large_count" -gt 0 ]; then
        record WARN "Large files in /tmp" "$large_count files > 100MB"
    else
        record OK "Large files in /tmp" "None"
    fi

    # /var/log total size
    local log_size
    log_size=$(du -sm /var/log 2>/dev/null | cut -f1 || echo "0")
    if [ "$log_size" -gt 1024 ]; then
        record WARN "/var/log size" "${log_size} MB (> 1 GB)"
    else
        record OK "/var/log size" "${log_size} MB"
    fi
}

# --- Summary Report ---

generate_summary() {
    local end_time
    end_time=$(date +%s)
    local elapsed=$((end_time - START_TIME))
    local total=$((CHECKS_PASSED + CHECKS_WARNED + CHECKS_FAILED))

    local overall="HEALTHY"
    [ "$CHECKS_WARNED" -gt 0 ] && overall="DEGRADED"
    [ "$CHECKS_FAILED" -gt 0 ] && overall="UNHEALTHY"

    section "Summary"
    echo ""
    echo "  Status:   $overall"
    echo "  Passed:   $CHECKS_PASSED / $total"
    echo "  Warnings: $CHECKS_WARNED"
    echo "  Failed:   $CHECKS_FAILED"
    echo "  Duration: ${elapsed}s"
    echo "  Time:     $(date '+%Y-%m-%d %H:%M:%S')"
    echo ""
}

# --- Main ---

main() {
    echo "=========================================="
    echo "  Health Check Suite"
    echo "  Host: $(hostname)"
    echo "  Date: $(date '+%Y-%m-%d %H:%M:%S')"
    echo "=========================================="

    check_system_resources
    check_connectivity
    check_processes
    check_filesystem
    generate_summary
}

main "$@"
```

```bash
cat > ~/backup-system.sh << 'SCRIPT'
#!/bin/bash
set -uo pipefail

# Automated Backup System
# Features:
#   - Full and incremental backups with rsync + hard links
#   - Configurable retention policy
#   - Backup verification with spot checks
#   - Email notification simulation
#   - Comprehensive logging
#   - Lock file to prevent concurrent runs

BACKUP_ROOT="/tmp/backup-system-demo"
SOURCE_DIR="/etc"
BACKUP_DIR="$BACKUP_ROOT/backups"
LOG_FILE="$BACKUP_ROOT/backup.log"
STATE_FILE="$BACKUP_ROOT/.state"
NOTIFICATION_LOG="$BACKUP_ROOT/notifications.log"
LOCK_FILE="$BACKUP_ROOT/.lock"
RETENTION_DAYS=7

# --- Setup ---

cleanup_demo() { rm -rf "$BACKUP_ROOT"; }
trap cleanup_demo EXIT
mkdir -p "$BACKUP_DIR" "$(dirname "$LOG_FILE")"

# --- Logging ---

log() {
    local level="$1"; shift
    local msg="[$(date '+%Y-%m-%d %H:%M:%S')] [$level] $*"
    echo "$msg"
    echo "$msg" >> "$LOG_FILE"
}

# --- Lock file ---

acquire_lock() {
    mkdir -p "$(dirname "$LOCK_FILE")"
    exec 200>"$LOCK_FILE"
    if ! flock -n 200; then
        log ERROR "Another backup is already running"
        return 1
    fi
    log INFO "Lock acquired (PID: $$)"
}

# --- Notification ---

send_notification() {
    local subject="$1" body="$2" severity="${3:-info}"
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [$severity] $subject" >> "$NOTIFICATION_LOG"
    echo "  $body" >> "$NOTIFICATION_LOG"
    # Production: mail -s "$subject" ops@company.com <<< "$body"
    # Production: curl -X POST "$SLACK_WEBHOOK" -d '{"text":"'"$body"'"}'
    log INFO "Notification sent: $subject"
}

# --- Full Backup ---

do_full_backup() {
    local timestamp
    timestamp=$(date '+%Y%m%d_%H%M%S')
    local backup_path="$BACKUP_DIR/full_$timestamp"

    log INFO "========================================="
    log INFO "Starting FULL backup"
    log INFO "  Source:      $SOURCE_DIR"
    log INFO "  Destination: $backup_path"
    log INFO "========================================="

    mkdir -p "$backup_path"
    local start_time
    start_time=$(date +%s)

    local rsync_output
    if rsync_output=$(rsync -a --stats "$SOURCE_DIR/" "$backup_path/" 2>&1); then
        local elapsed=$(( $(date +%s) - start_time ))
        local file_count
        file_count=$(echo "$rsync_output" | grep "Number of regular files transferred" | awk '{print $NF}' || echo "N/A")
        local total_size
        total_size=$(du -sh "$backup_path" | cut -f1)

        log INFO "Full backup complete in ${elapsed}s"
        log INFO "  Files: $file_count"
        log INFO "  Size:  $total_size"

        # Save state
        cat > "$STATE_FILE" << EOF
LAST_FULL=$backup_path
LAST_FULL_TIME=$timestamp
LAST_BACKUP=$backup_path
LAST_BACKUP_TYPE=full
EOF

        send_notification "Backup Complete: Full" \
            "Full backup of $SOURCE_DIR completed in ${elapsed}s ($total_size)" "success"

        echo "$backup_path"
        return 0
    else
        log ERROR "Full backup FAILED"
        send_notification "BACKUP FAILED" \
            "Full backup of $SOURCE_DIR failed! Check logs." "critical"
        return 1
    fi
}

# --- Incremental Backup ---

do_incremental_backup() {
    if [ ! -f "$STATE_FILE" ]; then
        log WARN "No previous state found — performing full backup"
        do_full_backup
        return $?
    fi

    source "$STATE_FILE"

    if [ -z "${LAST_FULL:-}" ] || [ ! -d "${LAST_FULL:-}" ]; then
        log WARN "Last full backup not found — performing full backup"
        do_full_backup
        return $?
    fi

    local timestamp
    timestamp=$(date '+%Y%m%d_%H%M%S')
    local backup_path="$BACKUP_DIR/incr_$timestamp"

    log INFO "========================================="
    log INFO "Starting INCREMENTAL backup"
    log INFO "  Source:      $SOURCE_DIR"
    log INFO "  Destination: $backup_path"
    log INFO "  Reference:   $LAST_FULL"
    log INFO "========================================="

    mkdir -p "$backup_path"
    local start_time
    start_time=$(date +%s)

    if rsync -a --stats --link-dest="$LAST_FULL" "$SOURCE_DIR/" "$backup_path/" 2>&1; then
        local elapsed=$(( $(date +%s) - start_time ))
        local total_size
        total_size=$(du -sh "$backup_path" | cut -f1)

        log INFO "Incremental backup complete in ${elapsed}s"
        log INFO "  Disk usage: $total_size (hard links save space)"

        cat > "$STATE_FILE" << EOF
LAST_FULL=$LAST_FULL
LAST_FULL_TIME=${LAST_FULL_TIME:-unknown}
LAST_BACKUP=$backup_path
LAST_BACKUP_TYPE=incremental
EOF

        send_notification "Backup Complete: Incremental" \
            "Incremental backup completed in ${elapsed}s ($total_size)" "success"

        echo "$backup_path"
        return 0
    else
        log ERROR "Incremental backup FAILED"
        send_notification "BACKUP FAILED" \
            "Incremental backup of $SOURCE_DIR failed!" "critical"
        return 1
    fi
}

# --- Verify Backup ---

verify_backup() {
    local backup_path="$1"
    log INFO "Verifying backup: $backup_path"

    if [ ! -d "$backup_path" ]; then
        log ERROR "Backup directory not found"
        return 1
    fi

    local file_count
    file_count=$(find "$backup_path" -type f | wc -l)
    if [ "$file_count" -eq 0 ]; then
        log ERROR "Backup is empty!"
        return 1
    fi

    # Spot-check known files
    local spot_checks=("hostname" "passwd" "group" "shells")
    local passed=0
    for f in "${spot_checks[@]}"; do
        [ -f "$backup_path/$f" ] && passed=$((passed + 1))
    done

    # Compare against source
    local source_count
    source_count=$(find "$SOURCE_DIR" -type f 2>/dev/null | wc -l)

    log INFO "Verification results:"
    log INFO "  Files in backup: $file_count"
    log INFO "  Files in source: $source_count"
    log INFO "  Spot checks:     $passed / ${#spot_checks[@]}"

    if [ "$passed" -eq "${#spot_checks[@]}" ] && [ "$file_count" -gt 0 ]; then
        log INFO "Verification PASSED"
        return 0
    else
        log ERROR "Verification FAILED"
        return 1
    fi
}

# --- Retention ---

apply_retention() {
    log INFO "Applying retention: keep backups from last $RETENTION_DAYS days"
    local removed=0
    local kept=0

    for dir in "$BACKUP_DIR"/full_* "$BACKUP_DIR"/incr_*; do
        [ -d "$dir" ] || continue
        if find "$dir" -maxdepth 0 -mtime +"$RETENTION_DAYS" 2>/dev/null | grep -q .; then
            log INFO "  Removing expired: $(basename "$dir")"
            rm -rf "$dir"
            removed=$((removed + 1))
        else
            kept=$((kept + 1))
        fi
    done

    log INFO "Retention: kept $kept, removed $removed"
}

# --- List Backups ---

list_backups() {
    echo ""
    echo "=== Backup Inventory ==="
    printf "  %-12s %-22s %-10s %s\n" "TYPE" "TIMESTAMP" "SIZE" "PATH"
    printf "  %-12s %-22s %-10s %s\n" "----" "---------" "----" "----"

    for dir in "$BACKUP_DIR"/full_* "$BACKUP_DIR"/incr_*; do
        [ -d "$dir" ] || continue
        local name
        name=$(basename "$dir")
        local type="${name%%_*}"
        local ts="${name#*_}"
        local size
        size=$(du -sh "$dir" | cut -f1)
        printf "  %-12s %-22s %-10s %s\n" "$type" "$ts" "$size" "$dir"
    done
    echo ""
}

# --- Main ---

main() {
    log INFO ""
    log INFO "============================================"
    log INFO "  Automated Backup System"
    log INFO "  Source: $SOURCE_DIR"
    log INFO "  Dest:   $BACKUP_DIR"
    log INFO "============================================"

    acquire_lock || exit 1

    local overall_start
    overall_start=$(date +%s)

    # Step 1: Full backup
    local full_path
    full_path=$(do_full_backup) || {
        log ERROR "Full backup failed — aborting"
        exit 1
    }

    # Step 2: Verify full backup
    verify_backup "$full_path" || {
        send_notification "ALERT" "Backup verification failed" "critical"
    }

    # Step 3: Incremental backup (simulate a change)
    log INFO ""
    log INFO "(Simulating time passing for incremental demo...)"
    sleep 1

    local incr_path
    incr_path=$(do_incremental_backup) || {
        log ERROR "Incremental backup failed"
    }

    # Step 4: Verify incremental
    if [ -n "${incr_path:-}" ]; then
        verify_backup "$incr_path" || true
    fi

    # Step 5: Apply retention
    log INFO ""
    apply_retention

    # Step 6: Show inventory
    list_backups

    local total_elapsed=$(( $(date +%s) - overall_start ))

    log INFO "============================================"
    log INFO "  Backup run complete"
    log INFO "  Total time: ${total_elapsed}s"
    log INFO "============================================"

    # Show notifications
    echo "=== Notifications Sent ==="
    [ -f "$NOTIFICATION_LOG" ] && cat "$NOTIFICATION_LOG"
    echo ""

    echo "=== Backup Log (last 10 lines) ==="
    tail -10 "$LOG_FILE"
}

main "$@"
```

## Key Takeaways

- Use `: > file` to truncate logs instead of `rm`/recreate — preserves the inode for processes still writing
- Backup scripts need three things: the backup itself, verification, and retention cleanup
- `rsync --link-dest` makes incremental backups efficient by hard-linking unchanged files
- Atomic symlink deployment (`ln -sfn new tmp && mv -Tf tmp current`) prevents partial deploys
- Health checks should cover system resources, service ports, processes, and filesystem state
- Alert cooldowns prevent notification storms — do not page every 60 seconds for the same issue
- Every production script follows the same structure: config, utilities, core logic, main function
- Use `flock` to prevent concurrent runs of the same script (critical for cron jobs)
- Always simulate notifications during development — replace with real integrations later
- A backup that has not been verified is not a backup
