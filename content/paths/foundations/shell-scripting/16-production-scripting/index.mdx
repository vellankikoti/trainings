Your backup script ran perfectly for six months. Then two cron jobs triggered it simultaneously, corrupting the backup. A deployment script injected user input into an `eval` statement, and an attacker used it to get shell access. A cleanup script was not idempotent, and when it ran twice, it deleted production data. ## Argument Parsing with getopts

Stop parsing `$1`, `$2` manually. `getopts` handles flags, options, validation, and help text properly:

```bash
#!/bin/bash
set -euo pipefail

# Default values
ENVIRONMENT="staging"
VERSION=""
DRY_RUN=false
VERBOSE=false
FORCE=false

usage() {
    cat << EOF
Usage: $(basename "$0") [OPTIONS] -v VERSION

Deploy application to the specified environment.

Options:
    -e ENV      Target environment (default: staging)
    -v VERSION  Version to deploy (required)
    -d          Dry run — show what would happen without doing it
    -f          Force — skip confirmation prompts
    -V          Verbose output
    -h          Show this help message

Examples:
    $(basename "$0") -v 2.1.0                    # Deploy v2.1.0 to staging
    $(basename "$0") -e production -v 2.1.0 -d   # Dry run on production
    $(basename "$0") -e production -v 2.1.0 -f   # Force deploy to production
EOF
    exit "${1:-0}"
}

while getopts ":e:v:dfVh" opt; do
    case "$opt" in
        e) ENVIRONMENT="$OPTARG" ;;
        v) VERSION="$OPTARG" ;;
        d) DRY_RUN=true ;;
        f) FORCE=true ;;
        V) VERBOSE=true ;;
        h) usage 0 ;;
        :) echo "Error: -$OPTARG requires an argument" >&2; usage 1 ;;
        *) echo "Error: Unknown option -$OPTARG" >&2; usage 1 ;;
    esac
done
shift $((OPTIND - 1))

# Validate required arguments
if [ -z "$VERSION" ]; then
    echo "Error: -v VERSION is required" >&2
    usage 1
fi

# Validate environment
case "$ENVIRONMENT" in
    staging|production|development) ;;
    *) echo "Error: Invalid environment '$ENVIRONMENT'" >&2; exit 1 ;;
esac

echo "Deploy: version=$VERSION env=$ENVIRONMENT dry_run=$DRY_RUN"
```

<Callout type="tip" title="getopts Syntax">
The option string `:e:v:dfVh` means: leading `:` enables silent error handling, `e:` takes an argument, `v:` takes an argument, `d`, `f`, `V` are flags (no argument), `h` is a flag. If an option requires an argument but none is given, `opt` is set to `:` and `OPTARG` holds the option letter.
</Callout>

## Configuration File Loading

Hard-coded values do not belong in scripts. Load them from config files with validation:

```bash
#!/bin/bash
set -euo pipefail

CONFIG_FILE="${CONFIG_FILE:-/etc/myapp/config.conf}"

# Default values (overridden by config file)
APP_PORT=8080
APP_ENV="production"
DB_HOST="localhost"
DB_PORT=5432
DB_NAME="myapp"
LOG_LEVEL="info"
BACKUP_RETAIN_DAYS=30

load_config() {
    local config_file="$1"

    if [ ! -f "$config_file" ]; then
        echo "WARNING: Config file not found: $config_file (using defaults)" >&2
        return 0
    fi

    # Validate the config file contains only safe assignments
    # Reject lines with $(), backticks, pipes, semicolons
    if grep -qP '[\$`|;]' "$config_file"; then
        echo "ERROR: Config file contains unsafe characters" >&2
        echo "Only KEY=VALUE lines are allowed (no commands)" >&2
        return 1
    fi

    # Source only lines matching KEY=VALUE pattern
    while IFS='=' read -r key value; do
        # Skip comments and empty lines
        [[ "$key" =~ ^[[:space:]]*# ]] && continue
        [[ -z "${key// /}" ]] && continue

        # Trim whitespace
        key="${key// /}"
        value="${value## }"
        value="${value%% }"
        value="${value#\"}"
        value="${value%\"}"

        # Only allow known configuration keys
        case "$key" in
            APP_PORT|APP_ENV|DB_HOST|DB_PORT|DB_NAME|LOG_LEVEL|BACKUP_RETAIN_DAYS)
                declare -g "$key=$value"
                ;;
            *)
                echo "WARNING: Unknown config key ignored: $key" >&2
                ;;
        esac
    done < "$config_file"
}

validate_config() {
    local errors=0

    # Validate port numbers
    if ! [[ "$APP_PORT" =~ ^[0-9]+$ ]] || [ "$APP_PORT" -lt 1 ] || [ "$APP_PORT" -gt 65535 ]; then
        echo "ERROR: APP_PORT must be 1-65535 (got: $APP_PORT)" >&2
        errors=$((errors + 1))
    fi

    if ! [[ "$DB_PORT" =~ ^[0-9]+$ ]] || [ "$DB_PORT" -lt 1 ] || [ "$DB_PORT" -gt 65535 ]; then
        echo "ERROR: DB_PORT must be 1-65535 (got: $DB_PORT)" >&2
        errors=$((errors + 1))
    fi

    # Validate environment
    case "$APP_ENV" in
        production|staging|development) ;;
        *) echo "ERROR: Invalid APP_ENV: $APP_ENV" >&2; errors=$((errors + 1)) ;;
    esac

    # Validate log level
    case "$LOG_LEVEL" in
        debug|info|warn|error) ;;
        *) echo "ERROR: Invalid LOG_LEVEL: $LOG_LEVEL" >&2; errors=$((errors + 1)) ;;
    esac

    return "$errors"
}
```

## Structured Logging

Production scripts need timestamps, log levels, and output to both console and file:

```bash
#!/bin/bash

LOG_FILE="/var/log/myapp/script.log"
LOG_LEVEL="${LOG_LEVEL:-info}"

# Log level hierarchy: debug < info < warn < error
declare -A LOG_LEVELS=([debug]=0 [info]=1 [warn]=2 [error]=3)

log() {
    local level="$1"
    shift
    local message="$*"

    # Check if this level should be logged
    local level_num="${LOG_LEVELS[$level]:-1}"
    local threshold="${LOG_LEVELS[$LOG_LEVEL]:-1}"
    [ "$level_num" -lt "$threshold" ] && return 0

    local timestamp
    timestamp=$(date -u '+%Y-%m-%dT%H:%M:%SZ')
    local caller="${FUNCNAME[1]:-main}"
    local line="${BASH_LINENO[0]:-0}"

    # Structured log format (easily parsed by log aggregators)
    local entry="$timestamp level=$level caller=$caller line=$line msg=\"$message\""

    # Write to file (always)
    echo "$entry" >> "$LOG_FILE" 2>/dev/null

    # Write to console (with color)
    local color=""
    local reset='\033[0m'
    case "$level" in
        debug) color='\033[0;36m' ;;  # cyan
        info)  color='\033[0;32m' ;;  # green
        warn)  color='\033[0;33m' ;;  # yellow
        error) color='\033[0;31m' ;;  # red
    esac

    printf "${color}[%-5s]${reset} %s\n" "${level^^}" "$message" >&2
}

# Convenience functions
debug() { log debug "$@"; }
info()  { log info "$@"; }
warn()  { log warn "$@"; }
error() { log error "$@"; }
fatal() { log error "$@"; exit 1; }
```

## Lock Files: Preventing Concurrent Runs

Two copies of a backup script running at the same time corrupt data. Lock files prevent this:

```bash
#!/bin/bash
set -euo pipefail

LOCK_FILE="/var/run/myapp-backup.lock"

acquire_lock() {
    # Use a file descriptor lock (atomic, race-condition-free)
    exec 200>"$LOCK_FILE"

    if ! flock -n 200; then
        local other_pid
        other_pid=$(cat "$LOCK_FILE" 2>/dev/null || echo "unknown")
        echo "ERROR: Another instance is already running (PID: $other_pid)" >&2
        exit 1
    fi

    # Write our PID to the lock file for debugging
    echo $$ > "$LOCK_FILE"
    echo "Lock acquired (PID: $$)"
}

release_lock() {
    flock -u 200 2>/dev/null
    rm -f "$LOCK_FILE"
    echo "Lock released"
}

# Acquire lock and ensure cleanup
acquire_lock
trap release_lock EXIT

echo "Running protected operation..."
sleep 2
echo "Done"
```

<Callout type="warning" title="Never Use PID Files Alone">
A common but broken pattern is writing `$$` to a file and checking if that PID exists. This has race conditions: the script could crash after writing the PID file but before cleanup, leaving a stale lock. Use `flock` for atomic file locking — it is the correct approach.
</Callout>

## Idempotency

An idempotent script produces the same result whether you run it once or ten times. This is critical for scripts that run on schedules or get retried:

```bash
#!/bin/bash
set -euo pipefail

# BAD: Not idempotent — running twice creates duplicate entries
# echo "export PATH=/opt/app/bin:\$PATH" >> ~/.bashrc

# GOOD: Idempotent — check before adding
add_to_path() {
    local path_entry="$1"
    local profile_file="$2"

    if ! grep -qF "$path_entry" "$profile_file" 2>/dev/null; then
        echo "export PATH=$path_entry:\$PATH" >> "$profile_file"
        echo "Added $path_entry to $profile_file"
    else
        echo "$path_entry already in $profile_file — skipping"
    fi
}

# BAD: Not idempotent — fails if user already exists
# useradd deploy

# GOOD: Idempotent — check first
ensure_user() {
    local username="$1"
    if id "$username" &>/dev/null; then
        echo "User $username already exists"
    else
        useradd -m -s /bin/bash "$username"
        echo "Created user $username"
    fi
}

# BAD: Not idempotent — appends duplicate iptables rules
# iptables -A INPUT -p tcp --dport 80 -j ACCEPT

# GOOD: Idempotent — check if rule exists
ensure_firewall_rule() {
    local port="$1"
    if iptables -C INPUT -p tcp --dport "$port" -j ACCEPT 2>/dev/null; then
        echo "Firewall rule for port $port already exists"
    else
        iptables -A INPUT -p tcp --dport "$port" -j ACCEPT
        echo "Added firewall rule for port $port"
    fi
}

# BAD: Not idempotent — mkdir fails if directory exists
# mkdir /opt/app/data

# GOOD: Idempotent
mkdir -p /opt/app/data  # -p makes it idempotent
```

<Exercise number={1} title="Production-Ready CLI Tool">

```bash
cat > ~/prod-tool.sh << 'SCRIPT'
#!/bin/bash
set -euo pipefail

# A production-grade CLI tool demonstrating all best practices

SCRIPT_NAME=$(basename "$0")
SCRIPT_DIR=$(cd "$(dirname "$0")" && pwd)
SCRIPT_VERSION="1.0.0"

# --- Defaults ---
CONFIG_FILE=""
LOG_FILE="/tmp/prod-tool.log"
LOG_LEVEL="info"
DRY_RUN=false
VERBOSE=false
LOCK_FILE="/tmp/prod-tool.lock"

# --- Logging ---
declare -A LOG_LEVELS=([debug]=0 [info]=1 [warn]=2 [error]=3)

log() {
    local level="$1"; shift
    local level_num="${LOG_LEVELS[$level]:-1}"
    local threshold="${LOG_LEVELS[$LOG_LEVEL]:-1}"
    [ "$level_num" -lt "$threshold" ] && return 0

    local ts
    ts=$(date '+%Y-%m-%d %H:%M:%S')
    local msg="[$ts] [${level^^}] $*"

    echo "$msg" >> "$LOG_FILE" 2>/dev/null

    case "$level" in
        error) echo "$msg" >&2 ;;
        warn)  echo "$msg" >&2 ;;
        *)     echo "$msg" ;;
    esac
}

info()  { log info "$@"; }
warn()  { log warn "$@"; }
error() { log error "$@"; }
debug() { log debug "$@"; }

# --- Lock file ---
acquire_lock() {
    exec 200>"$LOCK_FILE"
    if ! flock -n 200; then
        local pid
        pid=$(cat "$LOCK_FILE" 2>/dev/null || echo "?")
        error "Another instance is running (PID: $pid)"
        exit 1
    fi
    echo $$ > "$LOCK_FILE"
    debug "Lock acquired"
}

release_lock() {
    flock -u 200 2>/dev/null
    rm -f "$LOCK_FILE"
    debug "Lock released"
}

# --- Config loading ---
load_config() {
    local config_file="$1"

    if [ ! -f "$config_file" ]; then
        error "Config file not found: $config_file"
        return 1
    fi

    # Security: reject files with shell injection patterns
    if grep -qP '[\$`|;&]|\$\(' "$config_file"; then
        error "Config file contains unsafe characters: $config_file"
        return 1
    fi

    info "Loading config: $config_file"
    while IFS='=' read -r key value; do
        [[ "$key" =~ ^[[:space:]]*# ]] && continue
        [[ -z "${key// /}" ]] && continue
        key="${key// /}"
        value="${value## }"
        value="${value%% }"
        value="${value#\"}"
        value="${value%\"}"

        case "$key" in
            LOG_LEVEL)   LOG_LEVEL="$value" ;;
            LOG_FILE)    LOG_FILE="$value" ;;
            TARGET_DIR)  TARGET_DIR="$value" ;;
            *)           debug "Unknown config key: $key" ;;
        esac
    done < "$config_file"
}

# --- Usage ---
usage() {
    cat << EOF
$SCRIPT_NAME v$SCRIPT_VERSION — Production-grade system maintenance tool

Usage:
    $SCRIPT_NAME [OPTIONS] COMMAND

Commands:
    check       Run system checks
    clean       Clean temporary files
    report      Generate system report

Options:
    -c FILE     Configuration file
    -l FILE     Log file (default: $LOG_FILE)
    -L LEVEL    Log level: debug, info, warn, error (default: $LOG_LEVEL)
    -d          Dry run
    -v          Verbose (sets log level to debug)
    -h          Show this help
    -V          Show version

Examples:
    $SCRIPT_NAME check
    $SCRIPT_NAME -c /etc/myapp.conf -d clean
    $SCRIPT_NAME -v report
EOF
    exit "${1:-0}"
}

# --- Commands ---

cmd_check() {
    info "=== System Checks ==="

    local checks_passed=0
    local checks_failed=0

    # Disk space check
    local disk_pct
    disk_pct=$(df / | awk 'NR==2 {print $5}' | tr -d '%')
    if [ "$disk_pct" -lt 90 ]; then
        info "[PASS] Disk: ${disk_pct}% used"
        checks_passed=$((checks_passed + 1))
    else
        warn "[FAIL] Disk: ${disk_pct}% used"
        checks_failed=$((checks_failed + 1))
    fi

    # Memory check
    local mem_available
    mem_available=$(awk '/MemAvailable/ {print int($2/1024)}' /proc/meminfo 2>/dev/null || echo "0")
    if [ "$mem_available" -gt 100 ]; then
        info "[PASS] Memory: ${mem_available}MB available"
        checks_passed=$((checks_passed + 1))
    else
        warn "[FAIL] Memory: ${mem_available}MB available"
        checks_failed=$((checks_failed + 1))
    fi

    # Load check
    local load
    load=$(awk '{print $1}' /proc/loadavg 2>/dev/null || echo "0")
    local cores
    cores=$(nproc 2>/dev/null || echo "1")
    info "[INFO] Load: $load (cores: $cores)"
    checks_passed=$((checks_passed + 1))

    # Temp directory writable
    if touch /tmp/.check-write-test 2>/dev/null; then
        rm -f /tmp/.check-write-test
        info "[PASS] /tmp is writable"
        checks_passed=$((checks_passed + 1))
    else
        warn "[FAIL] /tmp is not writable"
        checks_failed=$((checks_failed + 1))
    fi

    info ""
    info "Results: $checks_passed passed, $checks_failed failed"
    return "$checks_failed"
}

cmd_clean() {
    info "=== Cleaning Temporary Files ==="

    local cleaned=0

    # Clean old temp files (idempotent)
    local old_tmp_files
    old_tmp_files=$(find /tmp -maxdepth 1 -user "$(whoami)" -type f -mtime +7 2>/dev/null || true)

    if [ -n "$old_tmp_files" ]; then
        local count
        count=$(echo "$old_tmp_files" | wc -l)

        if $DRY_RUN; then
            info "[DRY RUN] Would remove $count old temp files"
            echo "$old_tmp_files" | head -5 | while read -r f; do
                info "  would remove: $f"
            done
        else
            echo "$old_tmp_files" | while read -r f; do
                rm -f "$f"
                debug "Removed: $f"
            done
            cleaned=$count
            info "Removed $cleaned old temp files"
        fi
    else
        info "No old temp files to clean"
    fi

    # Clean old log files (idempotent)
    if [ -f "$LOG_FILE" ]; then
        local log_size
        log_size=$(stat -c%s "$LOG_FILE" 2>/dev/null || echo "0")
        local max_size=$((10 * 1024 * 1024))  # 10MB

        if [ "$log_size" -gt "$max_size" ]; then
            if $DRY_RUN; then
                info "[DRY RUN] Would rotate log file ($((log_size / 1024 / 1024))MB)"
            else
                mv "$LOG_FILE" "${LOG_FILE}.old"
                touch "$LOG_FILE"
                info "Rotated log file ($((log_size / 1024 / 1024))MB)"
            fi
        fi
    fi

    info "Cleanup complete"
}

cmd_report() {
    info "=== System Report ==="
    echo ""
    echo "Report generated: $(date -u '+%Y-%m-%dT%H:%M:%SZ')"
    echo "Hostname: $(hostname)"
    echo "Kernel: $(uname -r)"
    echo "Uptime: $(uptime -p 2>/dev/null || uptime)"
    echo ""

    echo "--- Resources ---"
    printf "  %-12s %s\n" "CPU Cores:" "$(nproc 2>/dev/null || echo 'unknown')"
    printf "  %-12s %s\n" "Memory:" "$(free -h 2>/dev/null | awk '/Mem:/ {print $2}' || echo 'unknown')"
    printf "  %-12s %s\n" "Disk (/):" "$(df -h / | awk 'NR==2 {print $3 "/" $2 " (" $5 " used)"}')"
    echo ""

    echo "--- Top Processes by CPU ---"
    ps aux --sort=-%cpu 2>/dev/null | head -6 | awk '{printf "  %-12s %-6s %-6s %s\n", $1, $3"%", $4"%", $11}' || true
    echo ""

    echo "--- Top Processes by Memory ---"
    ps aux --sort=-%mem 2>/dev/null | head -6 | awk '{printf "  %-12s %-6s %-6s %s\n", $1, $4"%", $3"%", $11}' || true

    info "Report complete"
}

# --- Main ---

main() {
    # Parse options
    while getopts ":c:l:L:dvhV" opt; do
        case "$opt" in
            c) CONFIG_FILE="$OPTARG" ;;
            l) LOG_FILE="$OPTARG" ;;
            L) LOG_LEVEL="$OPTARG" ;;
            d) DRY_RUN=true ;;
            v) VERBOSE=true; LOG_LEVEL="debug" ;;
            h) usage 0 ;;
            V) echo "$SCRIPT_NAME v$SCRIPT_VERSION"; exit 0 ;;
            :) error "-$OPTARG requires an argument"; usage 1 ;;
            *) error "Unknown option: -$OPTARG"; usage 1 ;;
        esac
    done
    shift $((OPTIND - 1))

    # Load config if specified
    if [ -n "$CONFIG_FILE" ]; then
        load_config "$CONFIG_FILE" || exit 1
    fi

    # Get command
    local command="${1:-}"
    if [ -z "$command" ]; then
        error "No command specified"
        usage 1
    fi

    # Acquire lock
    acquire_lock
    trap release_lock EXIT

    info "Starting $SCRIPT_NAME v$SCRIPT_VERSION (command: $command)"
    $DRY_RUN && info "DRY RUN MODE — no changes will be made"

    # Dispatch command
    case "$command" in
        check)  cmd_check ;;
        clean)  cmd_clean ;;
        report) cmd_report ;;
        *)      error "Unknown command: $command"; usage 1 ;;
    esac

    info "$SCRIPT_NAME completed successfully"
}

main "$@"
SCRIPT

chmod +x ~/prod-tool.sh
echo "--- Help output ---"
~/prod-tool.sh -h

echo "--- Running check command ---"
~/prod-tool.sh check
echo ""
echo "--- Running clean in dry-run mode ---"
~/prod-tool.sh -d clean
echo ""
echo "--- Running report with verbose logging ---"
~/prod-tool.sh -v report
```

## Security Practices

### Never eval User Input

```bash
# DANGEROUS: User can inject arbitrary commands
# user_input="; rm -rf /"
# eval "echo $user_input"

# SAFE: Use variables directly, never through eval
echo "$user_input"

# If you must build dynamic commands, use arrays
cmd=("rsync" "-az" "--delete")
if [ "$VERBOSE" = true ]; then
    cmd+=("-v")
fi
cmd+=("$SOURCE" "$DEST")
"${cmd[@]}"  # Safe: each array element is a separate argument
```

### Secure Temporary Files

```bash
# DANGEROUS: Predictable temp file — race condition attack
# TEMP="/tmp/myapp.tmp"

# SAFE: Use mktemp for unpredictable temp files
TEMP=$(mktemp /tmp/myapp.XXXXXXXXXX)
TEMP_DIR=$(mktemp -d /tmp/myapp.XXXXXXXXXX)
trap 'rm -rf "$TEMP" "$TEMP_DIR"' EXIT

# SAFE: Use process substitution instead of temp files when possible
diff <(command1) <(command2)  # No temp files at all
```

### Restrictive Permissions

```bash
# Set secure umask at the start of scripts
umask 077  # New files: owner read/write only, no group/other access

# Validate file ownership before sourcing
validate_file_safety() {
    local file="$1"

    # Check owner is root or current user
    local owner
    owner=$(stat -c '%U' "$file")
    if [ "$owner" != "root" ] && [ "$owner" != "$(whoami)" ]; then
        echo "ERROR: $file is owned by $owner (expected root or $(whoami))" >&2
        return 1
    fi

    # Check no group/other write permission
    local perms
    perms=$(stat -c '%a' "$file")
    if [[ "$perms" =~ [2367][2367]$ ]]; then
        echo "ERROR: $file is group/world writable (permissions: $perms)" >&2
        return 1
    fi

    return 0
}
```

## When to Graduate to Python

Bash is the right tool until it is not. Here are the signals:

| Signal | Why Python Is Better |
|--------|---------------------|
| Script exceeds 300-500 lines | Python modules, classes, and packages organize large codebases |
| Complex data structures needed | Python has dictionaries, lists, sets, and classes |
| Heavy JSON/YAML/XML processing | `json`, `yaml`, `xml` modules vs fragile `jq`/`sed` parsing |
| REST API calls with auth, pagination | `requests` library vs complex `curl` chains |
| Error handling becomes unwieldy | Python exceptions with try/except/finally |
| Unit testing needed | `pytest` vs ad-hoc bash test scripts |
| Cross-platform requirement | Python runs identically on Linux, macOS, and Windows |
| Team has mixed experience levels | Python is more readable for most developers |

```bash
# If you find yourself writing this in bash...
# ...it's time for Python
result=$(curl -sf "https://api.example.com/v1/users" \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" | \
    jq -r '.data[] | select(.role == "admin") | .email' | \
    sort -u | \
    while read -r email; do
        # More curl calls, more jq parsing...
        echo "$email"
    done)
```

<Callout type="tip" title="The Hybrid Approach">
Many production systems use both: bash as the entry point and orchestrator (start services, check prerequisites, set environment), calling Python for the complex logic. A 20-line bash wrapper around a Python script gives you the best of both worlds.
</Callout>

<Exercise number={2} title="Idempotent Server Provisioning Script">

```bash
cat > ~/provision.sh << 'SCRIPT'
#!/bin/bash
set -euo pipefail

# Idempotent server provisioning script
# Safe to run multiple times — only applies changes that are needed

SCRIPT_NAME=$(basename "$0")
DRY_RUN="${DRY_RUN:-false}"
PROVISION_DIR="/tmp/provision-demo"

# Track changes
CHANGES_MADE=0
SKIPPED=0

log() {
    echo "[$(date '+%H:%M:%S')] $*"
}

changed() {
    log "[CHANGED] $*"
    CHANGES_MADE=$((CHANGES_MADE + 1))
}

skipped() {
    log "[OK]      $*"
    SKIPPED=$((SKIPPED + 1))
}

would_change() {
    log "[WOULD]   $*"
}

# --- Idempotent operations ---

ensure_directory() {
    local dir="$1"
    local owner="${2:-$(whoami)}"
    local perms="${3:-755}"

    if [ -d "$dir" ]; then
        skipped "Directory exists: $dir"
    else
        if $DRY_RUN; then
            would_change "Create directory: $dir"
        else
            mkdir -p "$dir"
            chmod "$perms" "$dir"
            changed "Created directory: $dir (perms: $perms)"
        fi
    fi
}

ensure_file() {
    local file="$1"
    local content="$2"
    local perms="${3:-644}"

    if [ -f "$file" ]; then
        local current
        current=$(cat "$file")
        if [ "$current" = "$content" ]; then
            skipped "File unchanged: $file"
            return 0
        else
            if $DRY_RUN; then
                would_change "Update file: $file"
            else
                echo "$content" > "$file"
                chmod "$perms" "$file"
                changed "Updated file: $file"
            fi
        fi
    else
        if $DRY_RUN; then
            would_change "Create file: $file"
        else
            echo "$content" > "$file"
            chmod "$perms" "$file"
            changed "Created file: $file (perms: $perms)"
        fi
    fi
}

ensure_line_in_file() {
    local file="$1"
    local line="$2"

    if [ ! -f "$file" ]; then
        if $DRY_RUN; then
            would_change "Create $file with line"
        else
            echo "$line" > "$file"
            changed "Created $file with line"
        fi
        return
    fi

    if grep -qF "$line" "$file"; then
        skipped "Line already in $file"
    else
        if $DRY_RUN; then
            would_change "Add line to $file"
        else
            echo "$line" >> "$file"
            changed "Added line to $file"
        fi
    fi
}

ensure_symlink() {
    local target="$1"
    local link_name="$2"

    if [ -L "$link_name" ]; then
        local current_target
        current_target=$(readlink -f "$link_name")
        local expected_target
        expected_target=$(readlink -f "$target" 2>/dev/null || echo "$target")

        if [ "$current_target" = "$expected_target" ]; then
            skipped "Symlink correct: $link_name -> $target"
            return
        fi
    fi

    if $DRY_RUN; then
        would_change "Create symlink: $link_name -> $target"
    else
        ln -sf "$target" "$link_name"
        changed "Created symlink: $link_name -> $target"
    fi
}

# --- Provisioning steps ---

provision_directories() {
    log ""
    log "=== Directories ==="
    ensure_directory "$PROVISION_DIR"
    ensure_directory "$PROVISION_DIR/app"
    ensure_directory "$PROVISION_DIR/app/config"
    ensure_directory "$PROVISION_DIR/app/logs"
    ensure_directory "$PROVISION_DIR/app/data"
    ensure_directory "$PROVISION_DIR/app/bin"
    ensure_directory "$PROVISION_DIR/app/tmp" "$(whoami)" "700"
}

provision_config() {
    log ""
    log "=== Configuration ==="

    ensure_file "$PROVISION_DIR/app/config/app.conf" \
"APP_PORT=8080
APP_ENV=production
LOG_LEVEL=info
MAX_WORKERS=4
DB_HOST=localhost
DB_PORT=5432" "640"

    ensure_file "$PROVISION_DIR/app/config/logging.conf" \
"log_dir=/var/log/myapp
max_size=100M
max_files=10
format=json" "644"
}

provision_scripts() {
    log ""
    log "=== Scripts ==="

    local start_script="#!/bin/bash
set -euo pipefail
echo \"Starting application...\"
exec /usr/bin/env node /opt/app/server.js"

    ensure_file "$PROVISION_DIR/app/bin/start.sh" "$start_script" "755"

    local stop_script="#!/bin/bash
set -euo pipefail
echo \"Stopping application...\"
kill -TERM \$(cat /var/run/myapp.pid 2>/dev/null) 2>/dev/null || true"

    ensure_file "$PROVISION_DIR/app/bin/stop.sh" "$stop_script" "755"
}

provision_links() {
    log ""
    log "=== Symlinks ==="
    ensure_symlink "$PROVISION_DIR/app" "$PROVISION_DIR/current"
}

provision_security() {
    log ""
    log "=== Security Hardening ==="

    ensure_line_in_file "$PROVISION_DIR/app/config/.env" \
        "# Security: restrict file creation permissions"

    ensure_file "$PROVISION_DIR/app/.gitignore" \
"*.log
*.pid
.env
config/secrets/
tmp/" "644"
}

# --- Main ---

main() {
    log "========================================="
    log "  Server Provisioning"
    log "  Target: $PROVISION_DIR"
    $DRY_RUN && log "  MODE: DRY RUN"
    log "========================================="

    provision_directories
    provision_config
    provision_scripts
    provision_links
    provision_security

    log ""
    log "========================================="
    log "  Provisioning Complete"
    log "  Changes: $CHANGES_MADE"
    log "  Already OK: $SKIPPED"
    $DRY_RUN && log "  (DRY RUN — no changes applied)"
    log "========================================="
}

main "$@"

# Demonstrate idempotency: run again
echo ""
echo "=== Running again to demonstrate idempotency ==="
echo ""
CHANGES_MADE=0
SKIPPED=0
main "$@"

# Cleanup
rm -rf "$PROVISION_DIR"
```

```bash
cat > ~/prod-init.sh << 'SCRIPT'
#!/bin/bash
set -euo pipefail

# Production initialization script
# Combines all production-grade patterns into one tool:
# - getopts argument parsing
# - Config file loading
# - Structured logging
# - Lock file protection
# - Idempotent operations
# - Dry run support
# - Clean signal handling

readonly SCRIPT_NAME=$(basename "$0")
readonly SCRIPT_VERSION="1.0.0"

# --- Defaults ---
ENVIRONMENT="staging"
CONFIG_FILE=""
LOG_FILE="/tmp/prod-init.log"
LOG_LEVEL="info"
DRY_RUN=false
LOCK_FILE="/tmp/prod-init.lock"
WORK_DIR="/tmp/prod-init-demo"

# --- Logging ---
declare -A LOG_LEVELS=([debug]=0 [info]=1 [warn]=2 [error]=3)

_log() {
    local level="$1"; shift
    local ln="${LOG_LEVELS[$level]:-1}"
    local thr="${LOG_LEVELS[$LOG_LEVEL]:-1}"
    [ "$ln" -lt "$thr" ] && return 0

    local ts
    ts=$(date -u '+%Y-%m-%dT%H:%M:%SZ')
    local entry="$ts level=$level pid=$$ msg=\"$*\""

    echo "$entry" >> "$LOG_FILE" 2>/dev/null

    local color="" reset='\033[0m'
    case "$level" in
        debug) color='\033[36m' ;; info) color='\033[32m' ;;
        warn) color='\033[33m' ;; error) color='\033[31m' ;;
    esac
    printf "${color}[%-5s]${reset} %s\n" "${level^^}" "$*"
}

debug() { _log debug "$@"; }
info()  { _log info "$@"; }
warn()  { _log warn "$@"; }
error() { _log error "$@"; }
fatal() { _log error "$@"; exit 1; }

# --- Lock file ---
acquire_lock() {
    exec 200>"$LOCK_FILE"
    if ! flock -n 200; then
        fatal "Another instance is running (lock: $LOCK_FILE)"
    fi
    echo $$ > "$LOCK_FILE"
    debug "Lock acquired"
}

# --- Config ---
load_config() {
    local f="$1"
    [ ! -f "$f" ] && { error "Config not found: $f"; return 1; }

    if grep -qP '[\$`|;&]' "$f"; then
        error "Unsafe characters in config: $f"
        return 1
    fi

    while IFS='=' read -r key value; do
        [[ "$key" =~ ^[[:space:]]*# ]] && continue
        [[ -z "${key// /}" ]] && continue
        key="${key// /}"; value="${value## }"; value="${value%% }"
        value="${value#\"}"; value="${value%\"}"

        case "$key" in
            ENVIRONMENT)  ENVIRONMENT="$value" ;;
            LOG_LEVEL)    LOG_LEVEL="$value" ;;
            WORK_DIR)     WORK_DIR="$value" ;;
            *) debug "Unknown config key: $key" ;;
        esac
    done < "$f"
    info "Config loaded: $f"
}

# --- Idempotent helpers ---
CHANGES=0
UNCHANGED=0

ensure_dir()  {
    if [ -d "$1" ]; then
        UNCHANGED=$((UNCHANGED + 1))
    elif $DRY_RUN; then
        info "[DRY] Would create: $1"
    else
        mkdir -p "$1"; CHANGES=$((CHANGES + 1))
        info "Created: $1"
    fi
}

ensure_file() {
    local file="$1" content="$2" perms="${3:-644}"
    if [ -f "$file" ] && [ "$(cat "$file")" = "$content" ]; then
        UNCHANGED=$((UNCHANGED + 1))
    elif $DRY_RUN; then
        info "[DRY] Would write: $file"
    else
        echo "$content" > "$file"; chmod "$perms" "$file"
        CHANGES=$((CHANGES + 1))
        info "Wrote: $file"
    fi
}

# --- Usage ---
usage() {
    cat << EOF
$SCRIPT_NAME v$SCRIPT_VERSION — Production initialization tool

Usage: $SCRIPT_NAME [OPTIONS]

Options:
    -e ENV      Environment: staging, production (default: staging)
    -c FILE     Configuration file
    -l LEVEL    Log level: debug, info, warn, error (default: info)
    -d          Dry run mode
    -h          Show help
    -V          Show version

Examples:
    $SCRIPT_NAME -e production
    $SCRIPT_NAME -c /etc/myapp.conf -d
    $SCRIPT_NAME -e staging -l debug
EOF
    exit "${1:-0}"
}

# --- Cleanup ---
cleanup() {
    flock -u 200 2>/dev/null
    rm -f "$LOCK_FILE"
    debug "Cleanup complete"
}

# --- Main init steps ---

init_structure() {
    info "--- Initializing directory structure ---"
    ensure_dir "$WORK_DIR"
    ensure_dir "$WORK_DIR/releases"
    ensure_dir "$WORK_DIR/shared/config"
    ensure_dir "$WORK_DIR/shared/logs"
    ensure_dir "$WORK_DIR/shared/data"
    ensure_dir "$WORK_DIR/shared/tmp"
}

init_config() {
    info "--- Initializing configuration ---"

    local port=8080
    [ "$ENVIRONMENT" = "production" ] && port=443

    ensure_file "$WORK_DIR/shared/config/app.conf" \
"# Auto-generated for $ENVIRONMENT
APP_ENV=$ENVIRONMENT
APP_PORT=$port
LOG_LEVEL=$LOG_LEVEL
WORKERS=$(nproc 2>/dev/null || echo 2)
GENERATED=$(date -u '+%Y-%m-%dT%H:%M:%SZ')" "640"

    ensure_file "$WORK_DIR/shared/config/logging.conf" \
"output=json
level=$LOG_LEVEL
file=$WORK_DIR/shared/logs/app.log
max_size=100M
rotate=7" "644"
}

init_scripts() {
    info "--- Initializing management scripts ---"

    ensure_file "$WORK_DIR/shared/healthcheck.sh" \
'#!/bin/bash
curl -sf http://localhost:${APP_PORT:-8080}/health > /dev/null 2>&1
exit $?' "755"

    ensure_file "$WORK_DIR/shared/rotate-logs.sh" \
'#!/bin/bash
find /var/log/myapp -name "*.log" -mtime +7 -delete 2>/dev/null
echo "Log rotation complete"' "755"
}

init_security() {
    info "--- Applying security settings ---"

    ensure_file "$WORK_DIR/.gitignore" \
"*.log
*.pid
.env
shared/config/secrets/
shared/tmp/" "644"

    # Set restrictive permissions on sensitive directories
    if ! $DRY_RUN; then
        chmod 700 "$WORK_DIR/shared/tmp" 2>/dev/null || true
        chmod 750 "$WORK_DIR/shared/config" 2>/dev/null || true
    fi
}

verify() {
    info "--- Verification ---"
    local errors=0

    for dir in "$WORK_DIR" "$WORK_DIR/releases" "$WORK_DIR/shared/config"; do
        if [ -d "$dir" ]; then
            debug "Verified: $dir"
        else
            error "Missing: $dir"
            errors=$((errors + 1))
        fi
    done

    if [ -f "$WORK_DIR/shared/config/app.conf" ]; then
        local env_check
        env_check=$(grep "APP_ENV=" "$WORK_DIR/shared/config/app.conf" | cut -d= -f2)
        if [ "$env_check" = "$ENVIRONMENT" ]; then
            info "Environment verified: $env_check"
        else
            error "Environment mismatch: expected $ENVIRONMENT, got $env_check"
            errors=$((errors + 1))
        fi
    fi

    return "$errors"
}

main() {
    # Parse arguments
    while getopts ":e:c:l:dhV" opt; do
        case "$opt" in
            e) ENVIRONMENT="$OPTARG" ;;
            c) CONFIG_FILE="$OPTARG" ;;
            l) LOG_LEVEL="$OPTARG" ;;
            d) DRY_RUN=true ;;
            h) usage 0 ;;
            V) echo "$SCRIPT_NAME v$SCRIPT_VERSION"; exit 0 ;;
            :) fatal "-$OPTARG requires an argument" ;;
            *) fatal "Unknown option: -$OPTARG" ;;
        esac
    done

    # Load config if provided
    [ -n "$CONFIG_FILE" ] && load_config "$CONFIG_FILE"

    # Signal handling + cleanup
    trap cleanup EXIT
    trap 'warn "Interrupted"; exit 1' SIGINT SIGTERM

    # Acquire lock
    acquire_lock

    info "========================================="
    info "  Production Init v$SCRIPT_VERSION"
    info "  Environment: $ENVIRONMENT"
    info "  Work dir:    $WORK_DIR"
    $DRY_RUN && info "  Mode:        DRY RUN"
    info "========================================="

    init_structure
    init_config
    init_scripts
    init_security

    if ! $DRY_RUN && verify; then
        info ""
        info "========================================="
        info "  Initialization COMPLETE"
        info "  Changes:   $CHANGES"
        info "  Unchanged: $UNCHANGED"
        info "========================================="
    elif $DRY_RUN; then
        info ""
        info "Dry run complete — no changes applied"
    else
        fatal "Verification failed"
    fi
}

main "$@"

echo ""
info "--- Re-running to prove idempotency ---"
CHANGES=0
UNCHANGED=0
DRY_RUN=false

init_structure
init_config
init_scripts
init_security
info "Second run: $CHANGES changes, $UNCHANGED unchanged"

# Cleanup demo files
rm -rf "$WORK_DIR" /tmp/prod-init*.lock /tmp/prod-init.log
SCRIPT
```

## Key Takeaways

- Use `getopts` for argument parsing — it handles flags, options, validation, and error messages correctly
- Load configuration from files, not hard-coded values; validate every key and reject unknown or unsafe content
- Structured logging with timestamps and levels makes debugging production issues possible
- Use `flock` for lock files — PID-file-only approaches have race conditions
- Write idempotent scripts that check state before making changes — safe to re-run any number of times
- Never use `eval` with user input; use arrays to build dynamic commands safely
- Use `mktemp` for temporary files; never use predictable paths in `/tmp`
- Set `umask 077` and validate file permissions in security-sensitive scripts
- Graduate to Python when scripts exceed 300-500 lines, need complex data structures, or heavy API interaction
- The hybrid approach works well: a small bash wrapper that sets up the environment and calls Python for complex logic
