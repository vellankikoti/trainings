<Callout type="story" title="The Story">
Your new web app won't start. The error says "port 8080 already in use." Something else is listening on that port. You need to find what process, decide if you can kill it or need to change your port, and understand why this happens. Ports are how the OS routes network traffic to the right application — and port conflicts are one of the most common DevOps headaches.
</Callout>

## What Is a Port?

An IP address identifies a machine. A **port** identifies a specific application or service on that machine. Together, they form a unique address for network communication.

```
IP Address:Port = Complete destination

192.168.1.100:80    → Web server on that machine
192.168.1.100:22    → SSH server on that machine
192.168.1.100:5432  → PostgreSQL on that machine
```

Think of it like a building address. The IP address is the street address. The port is the apartment number. Mail (packets) needs both to reach the right recipient.

```bash
# See ports in action — every connection has a source and destination port
ss -tan | head -10
# State    Recv-Q    Send-Q    Local Address:Port    Peer Address:Port
# ESTAB    0         0         192.168.1.10:22       192.168.1.5:54321
#                              ^server port (22)      ^client port (54321)
```

## Port Number Ranges

Port numbers range from 0 to 65535. They are divided into three ranges:

| Range | Name | Description |
|-------|------|-------------|
| 0-1023 | Well-known ports | Reserved for standard services (requires root to bind) |
| 1024-49151 | Registered ports | Assigned to common applications (no root needed) |
| 49152-65535 | Dynamic/Ephemeral | Temporary ports assigned by the OS for client connections |

```bash
# Check the ephemeral port range on your system
sysctl net.ipv4.ip_local_port_range 2>/dev/null
# Typically: 32768 - 60999

# When you make an outbound connection, the OS picks from this range
# for the source port
curl -s http://example.com > /dev/null &
ss -tan | grep "example.com" 2>/dev/null
# Your source port will be something like 48234 (ephemeral)
```

<Callout type="tip" title="Why Root for Well-Known Ports?">
Ports below 1024 require root privileges to bind. This is a security measure — it prevents unprivileged users from starting a rogue service that impersonates SSH (22) or HTTP (80). In containers and modern setups, you often use capabilities (`CAP_NET_BIND_SERVICE`) instead of full root access.
</Callout>


## What Is a Socket?

A **socket** is an endpoint for communication — the combination of an IP address and a port:

```
Socket = IP Address + Port

Examples:
  192.168.1.10:80     → a socket
  10.0.0.5:3306       → a socket
  127.0.0.1:22        → a socket
  0.0.0.0:443         → a socket (all interfaces)
```

## Socket Pairs

Every network connection is uniquely identified by a **socket pair** — the source socket and the destination socket:

```
Socket Pair = (Source IP:Port) + (Destination IP:Port)

Connection 1: (192.168.1.5:48234) → (93.184.216.34:80)
Connection 2: (192.168.1.5:48235) → (93.184.216.34:80)
Connection 3: (192.168.1.5:48236) → (10.0.0.5:5432)
```

This is how a web server handles thousands of connections on the same port. The port is 80 for all of them, but each connection has a different source IP:port, making each socket pair unique.

```bash
# See socket pairs in action
ss -tan state established
# Each line shows a unique socket pair:
# Local Address:Port    Peer Address:Port
# 192.168.1.10:80       192.168.1.5:48234
# 192.168.1.10:80       192.168.1.5:48235
# Same local port (80), different remote ports = different connections
```

## Listening vs Established

A socket can be in different states:

```bash
# LISTEN — waiting for incoming connections
# The service is running and ready to accept clients
ss -tlnp
# State    Local Address:Port
# LISTEN   0.0.0.0:22         → SSH accepting connections from any interface
# LISTEN   127.0.0.1:5432     → PostgreSQL only on localhost
# LISTEN   :::80              → Nginx on all IPv6 interfaces

# ESTABLISHED — active connection between two endpoints
ss -tan state established
# Both sides have completed the handshake and are exchanging data
```

## Binding Addresses: 0.0.0.0 vs 127.0.0.1

The bind address determines **which network interfaces** a service listens on:

```
0.0.0.0:8080    → Listen on ALL interfaces (accessible from network)
127.0.0.1:8080  → Listen only on localhost (local access only)
192.168.1.10:8080 → Listen only on that specific interface
:::8080         → Listen on all IPv6 interfaces
::1:8080        → Listen only on IPv6 localhost
```

```bash
# Show listening services and their bind addresses
ss -tlnp

# Example output:
# LISTEN  0.0.0.0:22      ← SSH accessible from anywhere
# LISTEN  127.0.0.1:5432  ← PostgreSQL only from localhost
# LISTEN  0.0.0.0:80      ← Nginx accessible from anywhere
```

<Callout type="warning" title="Security: Binding Matters">
A service bound to `0.0.0.0` is accessible from the network. If Redis binds to `0.0.0.0:6379` without authentication, anyone on the network can connect. Always bind services to `127.0.0.1` unless they genuinely need network access. This is one of the most common security misconfigurations.
</Callout>

## Finding What Is Using a Port

This is the most common port debugging task:

```bash
# Method 1: ss (recommended — always available)
ss -tlnp | grep :8080
# LISTEN  0.0.0.0:8080  users:(("node",pid=1234,fd=3))

# Method 2: lsof (shows file descriptors)
lsof -i :8080 2>/dev/null
# COMMAND  PID  USER  FD  TYPE DEVICE SIZE/OFF NODE NAME
# node    1234  app   3u  IPv4 54321  0t0  TCP *:8080 (LISTEN)

# Method 3: fuser (shows PIDs)
fuser -n tcp 8080 2>/dev/null
# 8080/tcp:  1234

# Method 4: /proc filesystem
# If you know the PID, check what it is
ls -la /proc/1234/exe 2>/dev/null
cat /proc/1234/cmdline 2>/dev/null | tr '\0' ' '
```

## Resolving Port Conflicts

When you get "Address already in use":

```bash
# Step 1: Find what's using the port
ss -tlnp | grep :8080
# LISTEN  0.0.0.0:8080  users:(("nginx",pid=5678,fd=6))

# Step 2: Decide your action
# Option A: Kill the conflicting process
kill 5678          # graceful shutdown
kill -9 5678       # force kill (last resort)

# Option B: Change your application's port
# In your app config, use 8081 instead of 8080

# Option C: Wait for the socket to close
# If you see TIME_WAIT, the port will be available soon
ss -tan | grep :8080

# Option D: Allow port reuse (SO_REUSEADDR)
# Most server applications set this automatically
# In Python:
#   sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
```

### Common Causes of Port Conflicts

```bash
# 1. Previous instance didn't shut down cleanly
#    Check for zombie processes
ps aux | grep -E "defunct|<zombie>" | head -5

# 2. Service manager restarting the old instance
systemctl status nginx 2>/dev/null

# 3. Docker container mapping the same port
docker ps --format "table {{.Names}}\t{{.Ports}}" 2>/dev/null | head -10

# 4. TIME_WAIT from a recently closed connection
ss -tan state time-wait | grep :8080

# 5. Another service using the same default port
ss -tlnp | grep :8080
```

## Port Forwarding Basics

Port forwarding maps a port on one machine to a port on another:

```bash
# SSH local port forwarding
# Access remote PostgreSQL (5432) as if it were local
ssh -L 5432:db-server:5432 jump-host
# Now connect to localhost:5432 and it reaches db-server:5432

# SSH remote port forwarding
# Expose your local dev server to a remote machine
ssh -R 8080:localhost:3000 remote-server
# Now remote-server:8080 forwards to your localhost:3000

# SSH dynamic port forwarding (SOCKS proxy)
ssh -D 1080 remote-server
# Configure apps to use localhost:1080 as SOCKS proxy
```

## Ephemeral Ports

When a client connects to a server, the OS assigns a temporary **ephemeral port** for the client side of the connection:

```bash
# Your client connection uses a random high port
curl http://example.com &
ss -tan | grep ":80"
# 192.168.1.10:48234 → 93.184.216.34:80
#              ^ephemeral port assigned by OS

# Check your system's ephemeral port range
sysctl net.ipv4.ip_local_port_range 2>/dev/null

# Running out of ephemeral ports causes "Cannot assign requested address"
# This happens with very high connection rates
# Count current ephemeral port usage
ss -tan | awk '{print $4}' | grep -oP ':\K[0-9]+$' | \
    awk '$1 >= 32768 && $1 <= 60999' | wc -l
```

```bash
# 1. List all listening TCP services with process names
echo "=== Listening TCP Services ==="
ss -tlnp

# 2. List all listening UDP services
```

<Exercise number={2} title="Socket Communication">

Use netcat to create a client-server socket connection and understand socket pairs:

```bash
# --- Part 1: Create a TCP server and connect ---
echo "=== Part 1: TCP Client-Server ==="

# Start a TCP server listening on port 12345
nc -l 12345 &
SERVER_PID=$!
sleep 1

# Verify it's listening
echo "Server listening:"
ss -tlnp | grep 12345

# Connect to it from localhost
echo "Hello from client" | nc -q 1 localhost 12345 &
CLIENT_PID=$!
sleep 1

# Show the socket pair
echo ""
echo "Socket pairs (before connection closes):"
ss -tan | grep 12345
# Notice: same port (12345) but different source ports

wait $SERVER_PID $CLIENT_PID 2>/dev/null

# --- Part 2: Multiple connections to same port ---
echo ""
echo "=== Part 2: Multiple Connections ==="

# Start a server that accepts multiple connections
# (using a simple loop)
for i in 1 2 3; do
    nc -l 12346 > /dev/null &
done
sleep 1

# Connect multiple clients
for i in 1 2 3; do
    echo "Client $i" | nc -q 1 localhost 12346 &
done
sleep 2

echo "All connections use port 12346 but have unique socket pairs"

# --- Part 3: UDP socket ---
echo ""
echo "=== Part 3: UDP Socket ==="

# Start a UDP listener
nc -u -l 12347 &
UDP_SERVER=$!
sleep 1

echo "UDP server listening:"
ss -ulnp | grep 12347

# Send a UDP message
echo "UDP message" | nc -u -q 1 localhost 12347
sleep 1

kill $UDP_SERVER 2>/dev/null

# --- Part 4: Observe bind addresses ---
echo ""
echo "=== Part 4: Bind Address Demonstration ==="

# Listen on all interfaces
nc -l 0.0.0.0 12348 &
ALL_PID=$!
sleep 1

# Listen on localhost only
nc -l 127.0.0.1 12349 &
LOCAL_PID=$!
sleep 1

echo "Bind address comparison:"
ss -tlnp | grep -E "1234[89]"
# 0.0.0.0:12348 → accessible from network
# 127.0.0.1:12349 → localhost only

kill $ALL_PID $LOCAL_PID 2>/dev/null

echo ""
echo "Cleanup complete."
```

<MiniProject title="Port Scanner">

Build a script that checks which common service ports are open on a host:

```bash
cat > ~/port-scanner.sh << 'SCRIPT'
#!/bin/bash
# Port Scanner
# Checks common service ports on a target host

TARGET="${1:-127.0.0.1}"
TIMEOUT=2

RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
CYAN='\033[0;36m'
NC='\033[0m'

# Define well-known ports and their services
declare -A PORTS
PORTS=(
    [21]="FTP"
    [22]="SSH"
    [25]="SMTP"
    [53]="DNS"
    [80]="HTTP"
    [110]="POP3"
    [143]="IMAP"
    [443]="HTTPS"
    [993]="IMAPS"
    [995]="POP3S"
    [3000]="Node.js/Grafana"
    [3306]="MySQL"
    [5432]="PostgreSQL"
    [6379]="Redis"
    [8080]="HTTP-Alt"
    [8443]="HTTPS-Alt"
    [9090]="Prometheus"
    [9200]="Elasticsearch"
    [27017]="MongoDB"
)

# Custom ports from command line (optional second argument)
if [ -n "$2" ]; then
    IFS=',' read -ra CUSTOM_PORTS <<< "$2"
fi

echo "============================================="
echo "  Port Scanner"
echo "  Target: $TARGET"
echo "  Timeout: ${TIMEOUT}s per port"
echo "  Time: $(date)"
echo "============================================="

echo ""
echo "Scanning well-known service ports..."
echo ""

printf "${CYAN}%-8s %-20s %-10s${NC}\n" "PORT" "SERVICE" "STATUS"
printf "%-8s %-20s %-10s\n" "----" "-------" "------"

OPEN_COUNT=0
CLOSED_COUNT=0
OPEN_PORTS=()

# Scan well-known ports
for port in $(echo "${!PORTS[@]}" | tr ' ' '\n' | sort -n); do
    service="${PORTS[$port]}"

    if timeout "$TIMEOUT" bash -c "echo > /dev/tcp/$TARGET/$port" 2>/dev/null; then
        printf "${GREEN}%-8s %-20s %-10s${NC}\n" "$port" "$service" "OPEN"
        OPEN_COUNT=$((OPEN_COUNT + 1))
        OPEN_PORTS+=("$port/$service")
    else
        printf "%-8s %-20s %-10s\n" "$port" "$service" "closed"
        CLOSED_COUNT=$((CLOSED_COUNT + 1))
    fi
done

# Scan custom ports if provided
if [ ${#CUSTOM_PORTS[@]} -gt 0 ]; then
    echo ""
    echo "Scanning custom ports..."
    echo ""

    for port in "${CUSTOM_PORTS[@]}"; do
        port=$(echo "$port" | tr -d ' ')
        service="custom"

        if timeout "$TIMEOUT" bash -c "echo > /dev/tcp/$TARGET/$port" 2>/dev/null; then
            printf "${GREEN}%-8s %-20s %-10s${NC}\n" "$port" "$service" "OPEN"
            OPEN_COUNT=$((OPEN_COUNT + 1))
            OPEN_PORTS+=("$port/$service")
        else
            printf "%-8s %-20s %-10s\n" "$port" "$service" "closed"
            CLOSED_COUNT=$((CLOSED_COUNT + 1))
        fi
    done
fi

# Summary
echo ""
echo "============================================="
echo "  Scan Results"
echo "============================================="
echo "  Ports scanned: $((OPEN_COUNT + CLOSED_COUNT))"
echo -e "  Open:   ${GREEN}${OPEN_COUNT}${NC}"
echo -e "  Closed: ${CLOSED_COUNT}"

if [ ${#OPEN_PORTS[@]} -gt 0 ]; then
    echo ""
    echo "  Open ports:"
    for entry in "${OPEN_PORTS[@]}"; do
        echo "    - $entry"
    done
fi

# Security recommendations
echo ""
echo "--- Security Notes ---"

for entry in "${OPEN_PORTS[@]}"; do
    port="${entry%%/*}"
    case "$port" in
        6379)
            echo -e "${YELLOW}[WARN]${NC} Redis ($port) is open. Ensure authentication is enabled"
            echo "       and binding is restricted (not 0.0.0.0 in production)."
            ;;
        3306)
            echo -e "${YELLOW}[WARN]${NC} MySQL ($port) is open. Ensure it's not exposed to the internet."
            echo "       Bind to 127.0.0.1 if only local apps need access."
            ;;
        5432)
            echo -e "${YELLOW}[WARN]${NC} PostgreSQL ($port) is open. Check pg_hba.conf for access controls."
            ;;
        27017)
            echo -e "${YELLOW}[WARN]${NC} MongoDB ($port) is open. Ensure authentication is enabled."
            ;;
        9200)
            echo -e "${YELLOW}[WARN]${NC} Elasticsearch ($port) is open. Ensure X-Pack security is configured."
            ;;
        25)
            echo -e "${YELLOW}[WARN]${NC} SMTP ($port) is open. Ensure relay is properly restricted."
            ;;
    esac
done

echo ""
echo "Scan complete."
SCRIPT

chmod +x ~/port-scanner.sh

# Run the scanner against localhost
~/port-scanner.sh 127.0.0.1

# Example with custom ports:
# ~/port-scanner.sh 127.0.0.1 "8888,9999,4000"
```

## Key Takeaways

- **Ports** identify applications on a machine — IP gets to the machine, ports get to the app
- Three port ranges: **well-known** (0-1023, root required), **registered** (1024-49151), **ephemeral** (49152-65535)
- Memorize key ports: 22 (SSH), 53 (DNS), 80 (HTTP), 443 (HTTPS), 3306 (MySQL), 5432 (PostgreSQL), 6379 (Redis)
- A **socket** is an IP:port combination; a **socket pair** uniquely identifies a connection
- Use `ss -tlnp` to find what is listening; use `lsof -i :PORT` to find what process owns a port
- **Bind address matters**: `0.0.0.0` is network-accessible, `127.0.0.1` is local-only — always use the most restrictive option
- Port conflicts ("Address already in use") are solved by finding the conflicting process and deciding to kill it or pick a different port
- **SSH port forwarding** lets you securely access remote services through tunnels
